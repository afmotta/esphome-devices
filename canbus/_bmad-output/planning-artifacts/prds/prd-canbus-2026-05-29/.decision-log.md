# Decision Log — CAN Bus Smart Home PoC PRD

## 2026-05-29 — PRD initiated

**Intent:** Create (new PRD, no prior runs found)
**Scope decision:** PoC only — 2 nodes + 1 gateway, bench validation. Full deployment PRD deferred.
**Reason:** User wants to validate feasibility before hardware commitment at scale.

## 2026-05-29 — Gateway hardware change noted

**Decision:** PRD written against Waveshare ESP32-S3-RS485-CAN, not ESP32-S3-POE-ETH-8DI-8DO.
**Reason:** User confirmed actual PoC hardware is the RS485-CAN board.
**Impact:** No Ethernet (WiFi only), CAN on GPIO15/GPIO16 (not GPIO2/GPIO3), no PCA9554, no discrete I/O, no buzzer, no RGB LED. project-context.md references the POE board — that context applies to the future full deployment, not the PoC gateway.

## 2026-05-29 — CAT_OUTPUT deferred

**Decision:** Gateway → node commands (CAT_OUTPUT) out of scope for PoC.
**Reason:** User stated PoC goal is unidirectional validation (node → gateway → HA). Output commands are a future iteration.

## 2026-05-29 — Health monitoring minimized

**Decision:** Heartbeat events reaching HA (visible in Developer Tools) is the acceptance bar. No dashboard required.
**Reason:** PoC; user explicitly said "doesn't need to be elaborate, we can iterate."

## 2026-05-29 — Finalize complete
