- (decision) Join key: additive room_slug column on registry/nodes.csv (per ADR-0009 sec2 additive-column precedent), exported into map.json nodes[]. Reason: keeps canbus registry as single source of truth per ADR-0009 authority boundary; climate side never writes the registry.
- (decision) Ownership/validation: canbus registry (generate_nodes.py) owns and validates the room_slug join column -- every sensors=1 node's room_slug must match a known climate room_slug list at generation time. Reason: consistent with ADR-0009 authority boundary (HVAC reads, never writes registry).
- (direction) Alberto's direction on consumption mechanism: the HVAC (climate) firmware itself consumes CAN bus messages directly and exposes ESPHome template sensors per area/measurement, populated from decoded CAT_SENSOR frames -- NOT map.json-at-buildtime nor purely-via-HA-entities. Explicitly flagged by Alberto as needing evaluation of additional requirements and feasibility before being finalized.
- (direction) Alberto's direction on failover: evaluate whether the existing 3-tier failover (HA -> UDP -> Emergency) is still needed for CAN-sensed rooms, or whether direct CAN consumption can be the sole source of truth for those rooms. Not yet decided -- needs evaluation of CAN link reliability/staleness handling vs existing failover_sensor.yaml semantics.
- (constraint) Hardware check: climate master board (boards/waveshare-s3.yaml + waveshare-s3-ethernet.yaml, Waveshare ESP32-S3-POE-ETH-8DI-8RO) uses a W5500 SPI Ethernet chip on clk_pin=GPIO15, cs_pin=GPIO16 -- the exact two GPIOs the existing canbus gateway (a different physical board, Waveshare ESP32-S3-RS485-CAN) uses for its TWAI CAN transceiver (tx_pin=GPIO15, rx_pin=GPIO16 in canbus/firmware/gateway/gateway.yaml). The climate master cannot reuse the gateway's pin scheme; direct CAN consumption needs a new transceiver on different, currently-unconfirmed-free GPIOs, or a no-new-hardware relay path through the existing gateway.
- (note) Existing reusable staleness doctrine: canbus/firmware/protocol/node_health.h (ADR-0011 sec1, gateway-resident, header-only, already tested in test_node_health.cpp) implements '3 missed 30s heartbeats = LOST after 90s'. Same 30s cadence / 90s staleness contract is documented for CAT_SENSOR frames in sensor_kit.yaml. This doctrine is directly reusable as the built-in freshness/safety-net check if CAN becomes a primary or sole data source for climate rooms.
- (direction) Alberto: neither HVAC nor a separate 'Lighting system' are live yet, so the climate master controller board can be swapped. Candidate: Lilygo T-Connect PRO (native RS485 + CAN transceivers onboard), which would also replace GPIO-expander relays with RS485 Modbus relay boards (Waveshare Modbus RTU Relay 32CH) and keep Waveshare RTU Analog Output 8CH for 0-10V. Same controller model considered for the Lighting system's gateway too. Not yet a decision -- scope check needed before folding into this spec.
