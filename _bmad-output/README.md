# BMAD Output Directory

This directory contains all artifacts generated by the BMAD (Brownfield Management & Development) v6 framework for the ESPHome Devices project.

## Structure

```
_bmad-output/
├── planning-artifacts/       # Analysis, Planning & Architecture phase outputs
│   ├── prd.md               # Product Requirements Document
│   ├── product-brief.md     # Product vision and overview
│   ├── architecture.md      # System architecture documentation
│   ├── architecture-diagram.md  # Architecture diagrams
│   ├── brainstorming-session-results.md  # Initial brainstorming outputs
│   └── bmm-workflow-status.yaml  # Workflow progress tracking
│
└── implementation-artifacts/ # Implementation phase outputs
    ├── epic-*.md            # Epic briefs, completion reports, testing checklists
    ├── brief-*.md           # Feature-specific briefs
    └── brainstorming-*.md   # Feature-specific brainstorming sessions
```

## Directory Purposes

### planning-artifacts/

Contains outputs from the Analysis, Planning, and Solutioning phases of the BMAD methodology:

- **PRD**: Comprehensive product requirements including functional and non-functional requirements
- **Product Brief**: High-level product vision and problem statement
- **Architecture**: Detailed system architecture including component design, data flows, and technical decisions
- **Architecture Diagrams**: Visual representations using Mermaid diagrams
- **Brainstorming Sessions**: Structured ideation outputs from analysis phase
- **Workflow Status**: YAML file tracking progress through BMAD phases

### implementation-artifacts/

Contains outputs from the Implementation phase:

- **Epic Briefs**: Detailed specifications for each development epic
- **Completion Reports**: Post-implementation documentation of what was delivered
- **Testing Checklists**: QA validation criteria for each epic
- **Feature Briefs**: Specific feature planning documents
- **Feature Brainstorming**: Targeted ideation sessions for specific features

## Configuration

The directory structure is configured in `_bmad/bmm/config.yaml`:

```yaml
planning_artifacts: "{project-root}/_bmad-output/planning-artifacts"
implementation_artifacts: "{project-root}/_bmad-output/implementation-artifacts"
output_folder: "{project-root}/_bmad-output"
```

## Migration from v4 to v6

This project was migrated from BMAD v4 to v6. Key changes:

1. **New Directory Structure**: Introduced `_bmad-output/` with separate planning and implementation artifacts
2. **Workflow Status File**: Added `bmm-workflow-status.yaml` for phase tracking
3. **Document Organization**: Separated planning documents from implementation documents
4. **Preserved History**: All original documents from `docs/` were copied to maintain project history

## Working with BMAD Agents

BMAD agents reference these artifacts through the config variables:

- `{planning_artifacts}` → Points to planning-artifacts directory
- `{implementation_artifacts}` → Points to implementation-artifacts directory
- `{output_folder}` → Points to _bmad-output directory
