## Deferred from: code review of 3-2-gateway-cat-input-handler-button-events-to-home-assistant.md (2026-06-02)

- **No rate-limit/dedup on HA event firing** [`firmware/gateway.yaml:97`] — `homeassistant.event:` fires on every matching CAT_INPUT frame; a noisy bus or a stuck/repeating frame floods the HA event bus. Low risk for human-paced button clicks; pre-existing design, not introduced by Story 3.2. **Reviewed 2026-06-03 — do NOT add a gateway throttle/dedup.** Buttons are momentary ([[feedback_momentary_buttons_no_state]]): each gesture is one node-side `on_multi_click` event = one frame, and legitimate rapid distinct events (two single-clicks, different buttons) must all fire — a content-blind throttle would drop them. The real risk is a stuck/repeating frame, which is a hardware/bus fault better *surfaced* (heartbeat error flags) than silently muted. Folds into the command-reliability story below as fault detection, not rate-limiting.
