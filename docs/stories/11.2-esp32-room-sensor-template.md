### Template Substitutions Design

**Required Substitutions:**
```yaml
substitutions:
  device_name: "room-sensor-soggiorno"    # ESPHome device name (convention: room-sensor-{zone})
  zone_slug: "soggiorno"                   # Zone identifier for UDP IDs
  zone_name: "Soggiorno"                   # Display name for logs

**Optional Substitutions (with defaults):**
```yaml
  temp_offset: "0.0"     # Temperature calibration offset (°C)
  humidity_offset: "0.0" # Humidity calibration offset (%)
  i2c_sda: "21"          # I2C SDA pin (GPIO21 default for ESP32)
  i2c_scl: "22"          # I2C SCL pin (GPIO22 default for ESP32)
```

### SHT3x Wiring (I2C)

```
ESP32          SHT3x Sensor
-----          ------------
3.3V    ───────  VCC
GND     ───────  GND
GPIO21  ───────  SDA
GPIO22  ───────  SCL
```

**Alternative Pins for ESP32-C3:**
- SDA: GPIO8
- SCL: GPIO9

### UDP Broadcast Pattern

```yaml
udp:
  id: udp_packet_transport

packet_transport:
  - platform: udp
    udp_id: udp_packet_transport
    update_interval: 10s
    sensors:
      - id: temperature_sensor
        broadcast_id: "${zone_slug}_room_temp"
      - id: humidity_sensor
        broadcast_id: "${zone_slug}_room_humidity"
```

**Key Requirements:**
- Sensors must have `internal: true` (prevent HA duplication)
- Broadcast IDs must match receiver expectations
- Update interval matches Epic 9 pattern (10s)

### Hardware Selection Guide

**Recommended: ESP32-C3 + SHT31**
- **Cost:** $10-14 per sensor
- **Accuracy:** ±0.2°C temperature, ±2% RH humidity
- **Power:** USB-C 5V, 300mA typical
- **Compact:** Small form factor, easy mounting

**Alternative: ESP32-WROOM + DHT22**
- **Cost:** $12-17 per sensor
- **Accuracy:** ±0.5°C temperature, ±2% RH humidity
- **Power:** Micro-USB 5V, 500mA typical
- **Availability:** More common in retail

**Budget: ESP32-WROOM + DS18B20**
- **Cost:** $10-15 per sensor
- **Limitation:** Temperature only (no humidity)
- **Use Case:** Rooms without dew point concerns
