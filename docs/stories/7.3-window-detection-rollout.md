## Testing & Validation

### Per-Room Quick Validation
For each room with window detection enabled:
1. Open window (or toggle mock sensor)
2. Wait 3 minutes (timeout period)
3. Verify state transitions to "Window Open" in HA
4. Verify PID turns OFF (climate entity shows OFF mode)
5. Close window
6. Wait 3 minutes (recovery period)
7. Verify state returns to "Normal"
8. Verify PID resumes operation
9. Document results in `7.3-rollout-validation.md`

### System-Wide Regression Test
After all rooms deployed:
1. Monitor all room temperatures for 24 hours
2. Confirm no rooms fail to maintain setpoint (when windows closed)
3. Confirm no unexpected shutdowns (false positives)
4. Confirm diagnostic sensors visible and updating in HA
5. Confirm no errors in ESPHome device logs

## Risk Assessment

### Primary Risks

**Risk 1: Missing window sensors in HA**
- **Impact:** Cannot test window detection for affected rooms
- **Mitigation:** Document missing sensors, use mock sensors for testing
- **Resolution:** Deploy with `shutdown_modes: []` until physical sensors installed

**Risk 2: Multiple simultaneous OTA updates cause issues**
- **Impact:** Device becomes unresponsive or fails to update
- **Mitigation:** Deploy one device at a time, validate before next
- **Rollback:** Previous firmware binaries saved, can be reflashed via serial if needed

**Risk 3: Incorrect `shutdown_modes` configuration**
- **Impact:** Radiant room shuts down unnecessarily or fancoil room doesn't shut down
- **Mitigation:** Document equipment type for each room, validate configuration
- **Resolution:** Fix room configuration, redeploy device

**Risk 4: User confusion about new behavior**
- **Impact:** User thinks system is broken when window shutdown occurs
- **Mitigation:** Comprehensive documentation, diagnostic sensors for visibility
- **Resolution:** User education via window detection guide

### Rollback Plan

**If rollout causes system-wide issues:**
1. Identify affected rooms/devices
2. Remove window detection packages from affected room configurations
3. Recompile and OTA update affected devices
4. System returns to Epic 4 behavior (no window detection)
5. Investigate root cause, fix, and retry rollout incrementally

**If individual room has issues:**
1. Remove window detection from that room only
2. Redeploy device
3. Other rooms continue to benefit from window detection
4. Document issue for future resolution
