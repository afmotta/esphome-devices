# Story 13.8: PID Tuning for Cooling

**Epic:** 13 - Ground Floor Hybrid Radiant-Fancoil Cooling  
**Date:** December 26, 2025  
**Status:** Ready  
**Story Points:** 3  
**Priority:** High

---

## User Story

As a **system owner enabling radiant cooling**,  
I want **PID parameters tuned for cooling mode operation**,  
So that **radiant cooling responds appropriately without oscillation or overshoot**.

---

## Story Context

**Challenge:** Current radiant PID parameters are tuned for heating. Cooling has different thermal dynamics:
- **Heating:** Floor warms slowly, high thermal mass absorbs heat
- **Cooling:** Floor cools slowly, thermal mass releases heat gradually
- **Different response:** Cooling may require different kp/ki/kd values

**Risk from Epic Brief:**
> "PID tuning for cooling (High likelihood, Medium impact) - Plan dedicated tuning phase; start with conservative parameters"

**Zones to Tune:**
- Soggiorno
- Cucina
- Bagno
- Anticamera

## Technical Implementation

### Current Heating Parameters

**From existing room configs:**
```yaml
# Typical heating parameters
kp: 0.8
ki: 0.005
kd: 0.05
```

### Tuning Options

**Option A: ESPHome Autotune**

ESPHome PID supports autotune action:
```yaml
button:
  - platform: template
    name: "Autotune Radiant Soggiorno"
    on_press:
      - climate.pid.autotune: pid_radiant_soggiorno
```

**Considerations:**
- Autotune takes significant time (30+ minutes)
- Requires stable conditions during tuning
- May not work well for slow systems like radiant

**Option B: Manual Tuning (Ziegler-Nichols)**

1. Set ki = 0, kd = 0
2. Increase kp until sustained oscillation
3. Note critical gain (Kc) and oscillation period (Pc)
4. Calculate: kp = 0.6*Kc, ki = 2*kp/Pc, kd = kp*Pc/8

**Option C: Conservative Start (Recommended for MVP)**

1. Use heating parameters initially
2. If cooling is sluggish: increase kp by 10-20%
3. If oscillation occurs: decrease kp, increase ki
4. Monitor for 1-2 weeks, adjust as needed

### Suggested Conservative Starting Point

```yaml
# Cooling parameters (conservative start)
kp: 0.8    # Same as heating initially
ki: 0.003  # Slightly lower to prevent overshoot
kd: 0.05   # Same as heating
```

### Implementation Pattern

**If separate cooling parameters needed:**

```yaml
# In radiant.yaml, could add cooling-specific params
defaults:
  kp_heat: 0.8
  ki_heat: 0.005
  kd_heat: 0.05
  kp_cool: 0.8
  ki_cool: 0.003
  kd_cool: 0.05
```

**Note:** ESPHome PID doesn't natively support different parameters for heat/cool modes. May need to:
- Use same parameters for both (simplest)
- Switch parameters via lambda when mode changes (complex)
- Accept that conservative parameters work "well enough" for both

## Tuning Procedure

### Phase 1: Baseline (Week 1)

1. Deploy with heating parameters
2. Enable cooling mode
3. Set reasonable setpoints (e.g., 24°C)
4. Observe behavior:
   - How long to reach setpoint?
   - Any oscillation?
   - Any overshoot?

### Phase 2: Adjustment (Week 2)

Based on Phase 1 observations:

| Observation | Adjustment |
|-------------|------------|
| Too slow to cool | Increase kp by 10% |
| Oscillation | Decrease kp by 10%, increase ki |
| Overshoot | Decrease ki, increase kd |
| Good behavior | No change needed |

### Phase 3: Validation (Week 3)

1. Test various conditions:
   - Hot day (high cooling demand)
   - Mild day (low cooling demand)
   - Boost mode transitions
2. Fine-tune if needed
3. Document final parameters
