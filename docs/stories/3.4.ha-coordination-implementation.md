## Story

**I want** Home Assistant to coordinate climate modes and temperature sensor distribution across all boards,  
**so that** the system operates as a unified whole without board-to-board dependencies.

5. Test end-to-end: mode change in HA → all boards respond correctly
6. Test end-to-end: temperature changes → fancoils react appropriately
7. Document HA configuration and entity mappings
8. No manual intervention required for normal operation

### Architecture Context
**Home Assistant's New Role**: After Epic 3, HA becomes the central coordinator:
- **Temperature Distribution**: HA receives Dallas sensor data from gruppo-miscelazione and makes it available to distribution boards
- **Mode Coordination**: HA propagates climate mode changes (heat/cool/off) to all fancoils
- **Unified Control**: HA provides single interface for system-wide control
- **Failover**: HA can implement fallback logic if any board goes offline

**Key Entity Types**:
1. **Sensor entities**: `sensor.gruppo_miscelazione_dallas_piano_terra`, `sensor.gruppo_miscelazione_dallas_primo_piano`
2. **Climate entities**: `climate.fancoil_zona_1_pt`, `climate.fancoil_zona_2_pt`, ... (16 total fancoils)
3. **Mixing valve entities**: `climate.miscelatore_piano_terra`, `climate.miscelatore_primo_piano`

### Home Assistant Configuration Pattern

#### Example Automation: Mode Coordination
```yaml
automation:
  - alias: "Climate Mode - Heating"
    trigger:
      - platform: state
        entity_id: input_select.system_climate_mode
        to: "heating"
    action:
      - service: climate.set_hvac_mode
        target:
          entity_id: 
            - climate.fancoil_zona_1_pt
            - climate.fancoil_zona_2_pt
            # ... all 16 fancoils
        data:
          hvac_mode: heat
```

#### Example Climate Group
```yaml
climate:
  - platform: group
    name: "All Fancoils"
    entities:
      - climate.fancoil_zona_1_pt
      - climate.fancoil_zona_2_pt
      # ... all fancoils
```

### Important Considerations
1. **Sensor Update Frequency**: Ensure Dallas sensors update frequently enough for PID control
2. **Network Reliability**: Ethernet boards should be more reliable than WiFi
3. **HA Restart Behavior**: What happens when HA restarts? ESPHome boards should maintain last state
4. **Failover Strategy**: If gruppo-miscelazione goes offline, fancoils lose temperature data

### Testing

3. **Temperature sensing**: Fancoils respond to temperature changes
4. **Restart resilience**: System recovers correctly after HA or ESPHome restart
5. **Edge case handling**: Partial board offline, network interruption, etc.
