# Story 1.1 Testing Checklist - Integration Verification

This document provides step-by-step instructions for completing **Task 6: Integration verification testing**.

## Prerequisites

Before beginning testing:
- [ ] Both devices have hardware access (not required for pre-deployment checks)
- [ ] Home Assistant is accessible
- [ ] Backup configurations saved
- [ ] USB cable available for emergency serial recovery

## Pre-Deployment Testing (Can be done now - Task 5 subtasks)

### Step 1: Enable Feature Flag
1. Edit `locals/gruppo-miscelazione.yaml` (or use a test copy)
2. Change `use_modbus: "false"` to `use_modbus: "true"`
3. Run `esphome config locals/gruppo-miscelazione.yaml`
4. Verify: "Configuration is valid!" message

### Step 2: Firmware Size Verification
Current measurements (with use_modbus="false"):
- Master: RAM 10.8%, Flash 50.3% ✅
- Slave: RAM 11.0%, Flash 50.3% ✅

Verify sizes don't change significantly with use_modbus="true".

### Step 3: Entity ID Verification
Run this command to see all entity names/IDs in the master configuration:
```bash
esphome config locals/gruppo-miscelazione.yaml | grep -E "(name:|id:)" > /tmp/entities.txt
```

Check that existing entities are present:
- `relay_1` through `relay_6`
- `input_1` through `input_6`
- `dallas_0x81000000b3e6f628`
- `dallas_0xe9000000b366ed28`
- `pid_piano_terra_heat`, `pid_piano_terra_cool`
- `pid_primo_piano_heat`, `pid_primo_piano_cool`

New entities that should appear (only visible when use_modbus="true"):
- `Modbus Master Status`
- `Modbus Communication Errors`
- `Last Successful Modbus Poll`
- `Slave Test Register Value`

## Post-Deployment Testing (Requires hardware - Task 6)

**⚠️ IMPORTANT: Start with use_modbus="false" for safety**

### Phase 1: Baseline Validation (use_modbus="false")

1. **OTA Update Master Device**
   ```bash
   esphome upload locals/gruppo-miscelazione.yaml --device <IP_ADDRESS>
   ```
   - [ ] Upload completes successfully
   - [ ] Device reboots and reconnects to WiFi
   - [ ] Device appears online in Home Assistant

2. **OTA Update Slave Device**
   ```bash
   esphome upload locals/gruppo-distribuzione.yaml --device <IP_ADDRESS>
   ```
   - [ ] Upload completes successfully
   - [ ] Device reboots and reconnects to WiFi
   - [ ] Device appears online in Home Assistant

3. **Test Existing PID Controls** (Master - gruppo-miscelazione)
   - [ ] Open Home Assistant
   - [ ] Navigate to device entities
   - [ ] Change PID Piano Terra target temperature
   - [ ] Change PID Primo Piano target temperature
   - [ ] Verify controls respond normally

4. **Test Relay Outputs** (Master)
   - [ ] Toggle Relay 1 through Home Assistant
   - [ ] Toggle Relay 2 through Home Assistant
   - [ ] Verify physical relay clicks

5. **Test Relay Outputs** (Slave - distribuzione-piano-terra)
   - [ ] Toggle slow_pwm outputs through Home Assistant
   - [ ] Verify zones respond correctly

6. **Test Dallas Temperature Sensors** (Master)
   - [ ] Check `dallas_0x81000000b3e6f628` reports temperature
   - [ ] Check `dallas_0xe9000000b366ed28` reports temperature
   - [ ] Temperatures should be reasonable (10-40°C range)

7. **Verify WiFi Connectivity**
   - [ ] Check device signal strength in Home Assistant
   - [ ] Ping devices from network: `ping gruppo-miscelazione.local`
   - [ ] Verify no disconnections in ESPHome logs

8. **Verify I2C Devices** (Master - RTC, PCF8574)
   - [ ] Check if inputs 1-6 are responsive
   - [ ] Check RTC time is correct
   - [ ] Test PCF8574 relay outputs

9. **Monitor for 24-48 Hours**
   - [ ] No unexpected reboots
   - [ ] Temperature control remains stable (±0.5°C)
   - [ ] No errors in ESPHome logs

### Phase 2: Enable Modbus Testing (use_modbus="true")

**After Phase 1 passes successfully:**

1. **Enable Modbus on Both Devices**
   - Edit `locals/gruppo-miscelazione.yaml`: `use_modbus: "true"`
   - Edit `locals/gruppo-distribuzione.yaml`: `use_modbus: "true"`
   - Upload firmware to both devices

2. **Connect Serial Monitor to Master**
   ```bash
   esphome logs locals/gruppo-miscelazione.yaml --device <IP_ADDRESS>
   ```
   - [ ] Look for Modbus initialization messages
   - [ ] Check for successful Modbus controller startup

3. **Verify New Modbus Sensors Appear in Home Assistant**
   - [ ] `Modbus Master Status` = 1
   - [ ] `Modbus Communication Errors` = 0
   - [ ] `Last Successful Modbus Poll` shows timestamp
   - [ ] `Slave Test Register Value` appears (may show timeout initially)

4. **Test Simple Register Communication**
   - On slave device (distribuzione-piano-terra):
     - [ ] Find "Modbus Test Register" number entity in Home Assistant
     - [ ] Set it to a test value (e.g., 123)
   - On master device (gruppo-miscelazione):
     - [ ] Check "Slave Test Register Value" sensor
     - [ ] Verify it shows the same value (123)
     - [ ] This confirms master→slave communication works!

5. **Monitor Modbus Logs**
   ```bash
   esphome logs locals/gruppo-miscelazione.yaml --device <IP_ADDRESS> | grep -i modbus
   ```
   - [ ] Look for successful poll messages
   - [ ] Check for error messages (should be zero)
   - [ ] Verify communication every 10 seconds

6. **Verify No Regression**
   - [ ] All existing PID controls still function
   - [ ] All relay outputs still work
   - [ ] Temperature sensors still report
   - [ ] No impact on heating/cooling behavior
   - [ ] Home Assistant API remains stable

### Phase 3: Long-term Monitoring (24-48 hours)

- [ ] Device uptime stable (no unexpected reboots)
- [ ] Modbus communication errors remain at 0
- [ ] Temperature control accuracy unchanged (±0.5°C)
- [ ] All zones maintaining target temperatures
- [ ] CPU usage increase ≤ 5%
- [ ] No memory leaks (check free heap in logs)

## Rollback Procedure (If Issues Occur)

If any problems occur during testing:

1. **Immediate Rollback**
   - Set `use_modbus: "false"` in both locals files
   - Run OTA update to both devices
   - Verify system returns to normal operation

2. **Emergency Recovery**
   - Connect USB cable to device
   - Flash firmware via serial: `esphome run locals/<device>.yaml`
