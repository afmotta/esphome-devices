## PID Control Decision Tree Compliance

### Decision Path 1: Conditions Active/Recovering

- ✅ **Check:** `shutdown_required == true` (line 161)
- ✅ **PID not OFF:** Force to OFF via climate.control (lines 167-173)
- ✅ **PID already OFF:** No action, log maintenance (lines 176-178)

**Evidence:**
```cpp
if (shutdown_required) {
  if (pid->mode != climate::CLIMATE_MODE_OFF) {
    // Force PID OFF
    auto call = pid->make_call();
    call.set_mode(climate::CLIMATE_MODE_OFF);
    call.perform();
  } else {
    // Maintain shutdown
    ESP_LOGD("coordinator", "SHUTDOWN MAINTAINED...");
  }
}
```

- ✅ **Check:** `shutdown_required == false` (line 182)

## Edge Case Handling Compliance

### Edge Case 1: Simultaneous Triggers

- ✅ **Behavior:** Priority hierarchy resolves deterministically
- ✅ **Implementation:** Sequential if-checks with priority comparison (lines 126-153)
- ✅ **Result:** Emergency (priority=1) always wins over Window (priority=2)

### Edge Case 2: Recovery Interruption

- ✅ **Behavior:** New condition can become active during another's recovery
- ✅ **Implementation:** Priority comparison includes Recovering state (state==2)
- ✅ **Result:** PID stays OFF throughout transition

### Edge Case 3: Invalid Priority

- ✅ **Validation:** Check priority in range 1-99 (lines 129, 143)
- ✅ **Logging:** WARN message for invalid priority (lines 130, 144)
- ✅ **Graceful handling:** Ignore invalid condition, process remaining (lines 131, 145)

**Evidence:**
```cpp
if (emergency_priority < 1 || emergency_priority > 99) {
  ESP_LOGW("coordinator", "Invalid emergency priority %d, ignoring", emergency_priority);
} else {
  // Process valid priority
}
```
