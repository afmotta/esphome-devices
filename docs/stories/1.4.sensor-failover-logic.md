# Story 1.4: Sensor Failover Logic Implementation

## Status
Ready for Hardware Testing

## Story
**As a** distribution board,  
**I want** automatic failover between Modbus sensors and Home Assistant sensors,  
**so that** the system remains operational during communication failures.

## Acceptance Criteria

1. Create sensor selection logic in PID configurations:
   - Primary: Modbus sensor (if available and updated within 30 seconds)
   - Fallback: Home Assistant sensor (if available)
   - Emergency: Last known good value + safe shutdown after 5 minutes
2. Implement `template` sensor wrapper that selects best available source
3. Add configuration flag: `use_modbus: true/false` to enable/disable Modbus usage per device
4. Failover events logged and exposed as text sensor in Home Assistant:
   - "Modbus Active"
   - "Failover to Home Assistant"
   - "Emergency Mode - No Sensors"
5. Test scenarios:
   - Modbus communication failure → automatic HA failover
   - Home Assistant offline, Modbus active → system continues
   - Both offline → safe shutdown (turn off heating/cooling)
6. Recovery: When Modbus communication restored, automatic switch back from HA to Modbus

### Three-Tier Failover Architecture

**Tier Priority and Logic:**
```
┌─────────────────────────────────────────────────────────┐
│ Template Sensor (Exposed to PID Controllers)           │
│ ID: soggiorno_temperature_abstracted                    │
└────────────┬────────────────────────────────────────────┘
             │
             ├─ TIER 1 (Primary): Modbus Register Value
             │  ↳ If available && use_modbus=true && updated <30s → Use this
             │
             ├─ TIER 2 (Fallback): Home Assistant Sensor
             │  ↳ If Modbus unavailable && HA sensor available → Use HA
             │
             └─ TIER 3 (Emergency): NAN (Trigger Shutdown)
                ↳ If both unavailable for >5 min → Emergency mode
```

**Failover Logic Implementation (Template Sensor Lambda):**
```yaml
lambda: |-
  std::string flag = "${use_modbus}";
  
  // TIER 1: Modbus sensor (primary when enabled)
  if (flag == "true" && id(modbus_soggiorno_temperature).has_state()) {
    // Check if Modbus data is fresh (<30s old)
    if (id(modbus_soggiorno_temperature).state != NAN) {
      return id(modbus_soggiorno_temperature).state;
    }
  }
  
  // TIER 2: Home Assistant sensor (fallback)
  if (id(ha_soggiorno_temperature).has_state()) {
    return id(ha_soggiorno_temperature).state;
  }
  
  // TIER 3: Emergency - no sensors available
  return NAN;  // Triggers emergency shutdown after 5 min timeout
```

**Timeout Handling:**
- **Modbus Timeout:** 30 seconds (3 missed polls) — marks sensor unavailable
- **Failover Transition:** Immediate switch to HA when Modbus becomes unavailable
- **Emergency Timeout:** 5 minutes with NAN from all sources — trigger safe shutdown

### Template Sensor Design Pattern

**Component Structure (sensor_failover.yaml):**
```yaml
# Component: sensor_failover.yaml
# Purpose: Three-tier sensor failover wrapper for autonomous operation
# Required vars:
#   - sensor_id: Unique ID for failover sensor (e.g., "soggiorno_temperature")
#   - modbus_sensor_id: ID of Modbus sensor to use as primary
#   - ha_sensor_id: ID of Home Assistant sensor to use as fallback
#   - device_class: Sensor device class (temperature, humidity, etc.)
#   - unit_of_measurement: Unit string (e.g., "°C")
#   - accuracy_decimals: Decimal precision (default: 1)

# Template sensor with failover logic
sensor:
  - platform: template
    name: "${sensor_id}"
    id: ${sensor_id}_abstracted
    device_class: ${device_class}
    state_class: measurement
    unit_of_measurement: "${unit_of_measurement}"
    accuracy_decimals: ${accuracy_decimals}
    update_interval: 10s
    lambda: |-
      std::string flag = "${use_modbus}";
      
      if (flag == "true" && id(${modbus_sensor_id}).has_state()) {
        return id(${modbus_sensor_id}).state;
      } else if (id(${ha_sensor_id}).has_state()) {
        return id(${ha_sensor_id}).state;
      } else {
        return NAN;
      }

# Failover status tracking
text_sensor:
  - platform: template
    name: "${sensor_id} Source"
    id: ${sensor_id}_source
    lambda: |-
      std::string flag = "${use_modbus}";
      
      if (flag == "true" && id(${modbus_sensor_id}).has_state()) {
        return {"Modbus Active"};
      } else if (id(${ha_sensor_id}).has_state()) {
        return {"Failover to Home Assistant"};
      } else {
        return {"Emergency Mode - No Sensors"};
      }
    update_interval: 10s
```

**Usage in Device YAML:**
```yaml
packages:
  soggiorno_failover: !include
    file: ../components/sensor_failover.yaml
    vars:
      sensor_id: "soggiorno_temperature"
      modbus_sensor_id: "modbus_soggiorno_temperature"
      ha_sensor_id: "ha_soggiorno_temperature"
      device_class: "temperature"
      unit_of_measurement: "°C"
      accuracy_decimals: 1
```
