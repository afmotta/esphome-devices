## User Story

**As a** homeowner  
**I want** a reusable MEV component that exposes all hardware controls  
**So that** Home Assistant can orchestrate intelligent ventilation automation

## Description

Implement the reusable `components/mev.yaml` package that abstracts MEV hardware controls (4 relays, 0-10V fan speed, alarm input) as Home Assistant entities. This component follows the Epic 5 pattern: board exposes controls, HA provides intelligence.

## Technical Implementation Notes

### Component Interface
```yaml
# components/mev.yaml
# Expected vars:
# - mev_slug: "mev_primo_piano"
# - mev_name: "MEV Primo Piano"  
# - power_relay: output_1 (PCF8574 relay ID)
# - mode_relay: output_2
# - dehumid_relay: output_3
# - cooling_relay: output_4
# - fan_speed_dac: dac_1 (DAC output ID)
# - alarm_input: input_1 (binary sensor input)

switch:
  - platform: gpio
    name: "${mev_name} Power"
    id: "${mev_slug}_power"
    pin:
      pcf8574: outputs
      number: ${power_relay}
      
output:
  - platform: esp32_dac
    pin: GPIO25
    id: ${fan_speed_dac}
    
number:
  - platform: template
    name: "${mev_name} Fan Speed"
    id: "${mev_slug}_fan_speed"
    min_value: 0
    max_value: 100
    step: 1
    mode: slider
    lambda: "return 0;"
    set_action:
      - output.set_level:
          id: ${fan_speed_dac}
          level: !lambda 'return x / 100.0;'
```

### DAC Configuration
- KC868-A6 has built-in 0-10V DAC outputs (DAC_1/DAC_2)
- No external conversion needed
- Direct mapping: 0%→0V, 100%→10V (linear)
