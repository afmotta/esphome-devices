# Story 15.3: Floor-Wide Air Quality Aggregation

**Epic:** 15 - Air Quality Sensor Broadcasting  
**Date:** January 14, 2026  
**Status:** Done 
**Story Points:** 1  
**Priority:** High

---

## User Story

As a **climate-control board**,  
I want **floor-wide max CO₂ and max IAQ values computed locally**,  
So that **MEV control can respond to the worst air quality reading on the floor**.

---

## Story Context

**Existing Pattern:**

- First floor already uses `sensor.combination` with `type: max` for dew point aggregation
- This pattern proven effective for "worst case wins" scenarios
- MEV control needs single aggregate values, not 8 individual readings

**Design Decision (from Brainstorming):**

- Max-of-all approach: "worst room drives response"
- Ensures no room is neglected even if others have good air quality
- Simple, predictable, fail-safe behavior

### Implementation

**Add to climate-control.yaml sensor section:**
```yaml
sensor:
  # Floor-Wide Air Quality Aggregation (Max of all rooms)
  - platform: combination
    id: first_floor_max_co2
    name: "Max CO₂ Primo Piano"
    type: max
    unit_of_measurement: "ppm"
    device_class: "carbon_dioxide"
    state_class: "measurement"
    accuracy_decimals: 0
    sources:
      - source: bagno_grande_co2
      - source: bagno_ospiti_co2
      - source: bagno_padronale_co2
      - source: camera_nord_co2
      - source: camera_sud_co2
      - source: camera_ospiti_co2
      - source: camera_padronale_co2
      - source: lavanderia_co2

  - platform: combination
    id: first_floor_max_iaq
    name: "Max IAQ Primo Piano"
    type: max
    state_class: "measurement"
    accuracy_decimals: 0
    sources:
      - source: bagno_grande_iaq
      - source: bagno_ospiti_iaq
      - source: bagno_padronale_iaq
      - source: camera_nord_iaq
      - source: camera_sud_iaq
      - source: camera_ospiti_iaq
      - source: camera_padronale_iaq
      - source: lavanderia_iaq
```

### Aggregation Logic

| Sensor                | Sources         | Logic | Use Case                                        |
| --------------------- | --------------- | ----- | ----------------------------------------------- |
| `first_floor_max_co2` | 8 CO₂ receivers | MAX   | Drives fan speed; worst room = highest demand   |
| `first_floor_max_iaq` | 8 IAQ receivers | MAX   | Combined with CO₂ for AQ demand; higher = worse |

### Threshold Reference (for Epic 16)

| Metric      | Normal | Elevated | High      | Critical |
| ----------- | ------ | -------- | --------- | -------- |
| CO₂ (ppm)   | <800   | 800-1000 | 1000-1500 | >1500    |
| IAQ (index) | <100   | 100-200  | 200-300   | >300     |

## Notes

- Combination sensor `type: max` ignores NaN values automatically
- If all sources are NaN, combination sensor reports NaN (safe)
- These sensors are the foundation for Epic 16's AQ→Fan Speed mapping
- IAQ has no standard `device_class` in ESPHome, so omitted
- CO₂ 400ppm is outdoor baseline; >1500ppm indicates poor ventilation
- IAQ 0-50 is "Good", 51-100 is "Moderate", >150 is "Poor"
