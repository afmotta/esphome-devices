# Story 13.5: Fancoil Boost Coordinator Component

**Epic:** 13 - Ground Floor Hybrid Radiant-Fancoil Cooling  
**Date:** December 26, 2025  
**Status:** Ready  
**Story Points:** 5  
**Priority:** High (Core Feature)

---

## User Story

As a **system owner with hybrid radiant-fancoil cooling**,  
I want **an autonomous coordinator that switches between radiant-only and fancoil-boost modes**,  
So that **I get optimal comfort from radiant cooling with fancoil backup when needed**.

---

## Story Context

**Core Epic Deliverable:** This is the main coordination logic that implements the comfort-first hybrid cooling strategy.

**Operating Modes:**
| Mode | Radiant Floor | Fancoils | Trigger |
|------|---------------|----------|---------|
| **Normal Cooling** | PID active | OFF | Default |
| **Boost Cooling** | Fully open (100%) | PID active | delta > threshold |

**Key Requirements from Elicitation:**
- ESPHome-native (autonomous, works without HA)
- Hysteresis-based transitions (activate at threshold, deactivate at threshold/2)
- Minimum time-in-state protection (prevent rapid cycling)
- Single floor-wide threshold (from HA with fallback)

---

## Acceptance Criteria

### Mode Logic

1. **Boost Activation:**
   - [ ] Monitor room temperature vs setpoint
   - [ ] Activate boost when `(current_temp - target_temp) > threshold`
   - [ ] Use highest delta among hybrid rooms (Soggiorno, Cucina)

2. **Boost Deactivation:**
   - [ ] Deactivate when `(current_temp - target_temp) < (threshold / 2)`
   - [ ] Hysteresis prevents oscillation

3. **Minimum Time-in-State:**
   - [ ] Once in boost mode: stay minimum 10 minutes
   - [ ] Once in normal mode: stay minimum 10 minutes
   - [ ] Prevents valve wear and user confusion

### System Control

4. **Normal Mode Behavior:**
   - [ ] Radiant PIDs active (Soggiorno, Cucina, Bagno, Anticamera)
   - [ ] Fancoil PIDs OFF (Soggiorno, Cucina)
   - [ ] Zones controlled by individual room PIDs

5. **Boost Mode Behavior:**
   - [ ] Radiant zones fully open (slow_pwm at 100%)
   - [ ] Radiant PIDs effectively bypassed
   - [ ] Fancoil PIDs activated (COOL mode)
   - [ ] Fancoils control room temperature

6. **Mode Transitions:**
   - [ ] Log mode changes clearly
   - [ ] Smooth transition (no supply temp spikes)

### Diagnostic Sensors

7. **State Visibility:**
   - [ ] `text_sensor.ground_floor_cooling_mode` - "Radiant Only" / "Fancoil Boost"
   - [ ] `binary_sensor.fancoil_boost_active` - Boolean status
   - [ ] `sensor.ground_floor_max_delta` - Current highest delta (for debugging)

### Autonomous Operation

8. **HA Independence:**
   - [ ] Coordinator works during HA outage
   - [ ] Threshold falls back to default (4.0°C)
   - [ ] Mode decisions based on local sensor data

## Technical Implementation

### New Component File

**Create `components/fancoil_boost_coordinator.yaml`:**

```yaml
# Fancoil Boost Coordinator for Ground Floor
# Switches between radiant-only (comfort) and fancoil-boost (capacity) modes

defaults:
  min_time_in_state: 600  # seconds (10 minutes)
  default_threshold: 4.0  # °C

globals:
  - id: boost_mode_active
    type: bool
    initial_value: 'false'
  - id: last_mode_change_time
    type: uint32_t
    initial_value: '0'

binary_sensor:
  - platform: template
    id: fancoil_boost_active
    name: "Ground Floor Fancoil Boost Active"
    device_class: running
    lambda: 'return id(boost_mode_active);'

text_sensor:
  - platform: template
    id: ground_floor_cooling_mode
    name: "Ground Floor Cooling Mode"
    lambda: |-
      if (id(boost_mode_active)) {
        return {"Fancoil Boost"};
      }
      return {"Radiant Only"};
    update_interval: 5s

sensor:
  - platform: template
    id: ground_floor_max_delta
    name: "Ground Floor Max Temperature Delta"
    unit_of_measurement: "°C"
    accuracy_decimals: 1
    lambda: |-
      // Calculate max delta across hybrid rooms
      float soggiorno_delta = id(soggiorno_room_temp_abstracted).state - 
                              id(pid_radiant_soggiorno).target_temperature;
      float cucina_delta = id(cucina_room_temp_abstracted).state - 
                           id(pid_radiant_cucina).target_temperature;
      return max(soggiorno_delta, cucina_delta);
    update_interval: 30s

interval:
  - interval: 30s
    then:
      - lambda: |-
          // Get current state
          float threshold = id(effective_fancoil_boost_threshold).state;
          float hysteresis = threshold / 2.0;
          float max_delta = id(ground_floor_max_delta).state;
          uint32_t now = millis();
          uint32_t min_time = ${min_time_in_state} * 1000;
          
          // Check minimum time-in-state
          if ((now - id(last_mode_change_time)) < min_time) {
            ESP_LOGD("coordinator", "Min time not elapsed, staying in current mode");
            return;
          }
          
          // Only process during cooling mode
          if (!id(summer_mode).state) {
            return;
          }
          
          bool should_boost = false;
          bool should_normal = false;
          
          if (!id(boost_mode_active) && max_delta > threshold) {
            should_boost = true;
            ESP_LOGI("coordinator", "Boost activation: delta %.1f > threshold %.1f", 
                     max_delta, threshold);
          }
          
          if (id(boost_mode_active) && max_delta < hysteresis) {
            should_normal = true;
            ESP_LOGI("coordinator", "Normal activation: delta %.1f < hysteresis %.1f",
                     max_delta, hysteresis);
          }
          
          if (should_boost) {
            // Enter boost mode
            id(boost_mode_active) = true;
            id(last_mode_change_time) = now;
            
            // Set radiant zones to 100%
            id(slow_pwm_soggiorno).set_level(1.0);
            id(slow_pwm_cucina).set_level(1.0);
            id(slow_pwm_bagno_terra).set_level(1.0);
            id(slow_pwm_anticamera).set_level(1.0);
            
            // Enable fancoil PIDs
            auto call_s = id(pid_fancoil_soggiorno).make_call();
            call_s.set_mode("COOL");
            call_s.perform();
            
            auto call_c = id(pid_fancoil_cucina).make_call();
            call_c.set_mode("COOL");
            call_c.perform();
            
            ESP_LOGI("coordinator", "BOOST MODE ACTIVATED");
          }
          
          if (should_normal) {
            // Return to normal mode
            id(boost_mode_active) = false;
            id(last_mode_change_time) = now;
            
            // Disable fancoil PIDs
            auto call_s = id(pid_fancoil_soggiorno).make_call();
            call_s.set_mode("OFF");
            call_s.perform();
            
            auto call_c = id(pid_fancoil_cucina).make_call();
            call_c.set_mode("OFF");
            call_c.perform();
            
            // Radiant PIDs resume control automatically
            // (slow_pwm controlled by PID output)
            
            ESP_LOGI("coordinator", "NORMAL MODE ACTIVATED (Radiant Only)");
          }
```

### Integration Points

**Required IDs from other components:**
- `summer_mode` - Binary sensor for cooling season
- `effective_fancoil_boost_threshold` - From Story 13.4
- `soggiorno_room_temp_abstracted`, `cucina_room_temp_abstracted` - Room temps
- `pid_radiant_soggiorno`, `pid_radiant_cucina` - For target temps
- `pid_fancoil_soggiorno`, `pid_fancoil_cucina` - Fancoil control
- `slow_pwm_soggiorno`, `slow_pwm_cucina`, etc. - Radiant zone outputs

## Open Questions

1. **Radiant PID Behavior:** When `slow_pwm` is set manually to 100%, does the PID continue trying to adjust? Need to either disable PID or accept it will "fight" the 100% setting.

2. **Bagno/Anticamera in Boost:** Should these radiant-only rooms also go to 100% during boost? Current design says yes (maximize radiant capacity).

3. **Locale Tecnico:** Fancoil-only room - should it follow boost mode or operate independently?
