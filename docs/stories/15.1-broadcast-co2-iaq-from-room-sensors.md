# Story 15.1: Broadcast CO₂/IAQ from Room Sensors

**Epic:** 15 - Air Quality Sensor Broadcasting  
**Date:** January 14, 2026  
**Status:** Done  
**Story Points:** 2  
**Priority:** High

---

## User Story

As a **climate control system**,  
I want **CO₂ and IAQ data broadcast from room sensors via UDP**,  
So that **the climate-control board can make real-time MEV fan speed decisions without HA latency**.

---

## Story Context

**Existing Infrastructure:**

- Room sensors (S1-Pro-Multi-Sense) already broadcast temperature and humidity via UDP
- `packet_transport` platform configured in `boards/s1-pro-multi-sense.yaml`
- SCD40 CO₂ sensor available as `scd_co2` (id)
- BME688 IAQ sensor available as `iaq` (id)
- Climate-control board already receives temp/humidity from these providers

**Current packet_transport section:**
```yaml
# From boards/s1-pro-multi-sense.yaml
packet_transport:
  - platform: udp
    udp_id: udp_packet_transport
    update_interval: 10s
    sensors:
      - id: scd40_temp
        broadcast_id: "${room_slug}_room_temp"
      - id: scd40_rh
        broadcast_id: "${room_slug}_room_humidity"
```

### Implementation

**Modified packet_transport section:**
```yaml
# UDP Transport Configuration (for broadcasting sensor data)
udp:
  id: udp_packet_transport

# Packet Transport Configuration (broadcasts sensors to distribution boards)
packet_transport:
  - platform: udp
    udp_id: udp_packet_transport
    update_interval: 10s
    sensors:
      - id: scd40_temp
        broadcast_id: "${room_slug}_room_temp"
      - id: scd40_rh
        broadcast_id: "${room_slug}_room_humidity"
      - id: scd_co2
        broadcast_id: "${room_slug}_co2"
      - id: iaq
        broadcast_id: "${room_slug}_iaq"
```
