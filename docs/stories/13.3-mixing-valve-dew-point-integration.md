# Story 13.3: Mixing Valve Dew Point Integration

**Epic:** 13 - Ground Floor Hybrid Radiant-Fancoil Cooling  
**Date:** December 26, 2025  
**Status:** Ready  
**Story Points:** 2  
**Priority:** High (Safety Critical)

---

## User Story

As a **system owner enabling radiant cooling**,  
I want **the mixing valve to automatically limit supply water temperature based on dew point**,  
So that **condensation on floors is prevented during cooling operations**.

---

## Story Context

**Safety-Critical Feature:** This story implements the core condensation prevention mechanism. Supply water temperature must never drop below dew point + safety margin.

**Existing Pattern (First Floor):**
```yaml
# From components/rooms/first_floor/first-floor.yaml
sensor:
  - platform: combination
    id: first_floor_max_dew_point
    ...
    on_value:
      then:
        - climate.control:
            id: pid_mixing_valve_first_floor_radiant
            target_temperature: !lambda "return (x + ${safe_margin});"
```

**Ground Floor Mixing Valve:**
- Component: `mixing_pump_ground_floor_radiant` in `ground-floor.yaml`
- PID ID: `pid_mixing_valve_ground_floor_radiant`
- Controls supply water temperature via analog output

## Acceptance Criteria

### Functional Requirements

1. **Dew Point → Supply Temp Linkage:**
   - [ ] `ground_floor_max_dew_point` updates trigger supply temp adjustment
   - [ ] Supply temp minimum = `max_dew_point + safe_margin`
   - [ ] Default `safe_margin` = 1.0°C (configurable)

2. **Mixing Valve Control:**
   - [ ] `on_value` trigger on `ground_floor_max_dew_point` sensor
   - [ ] `climate.control` updates `pid_mixing_valve_ground_floor_radiant` target
   - [ ] Target = dew point + safe margin during cooling

3. **Mode Awareness:**
   - [ ] Dew point limiting only applies during cooling mode
   - [ ] Heating mode uses normal supply temp targets
   - [ ] Consider: How to detect cooling vs heating mode?

### Safety Requirements

4. **Always-On Protection:**
   - [ ] Dew point protection cannot be disabled accidentally
   - [ ] Protection active even if coordinator fails
   - [ ] Mixing valve respects limit regardless of zone states

5. **Failover Behavior:**
   - [ ] If `ground_floor_max_dew_point` unavailable → conservative fallback
   - [ ] Option A: Assume high dew point (e.g., 20°C)
   - [ ] Option B: Disable radiant cooling entirely
   - [ ] Document chosen approach

### Technical Requirements

6. **Variable Configuration:**
   - [ ] `safe_margin` as configurable default (1.0°C)
   - [ ] Can be adjusted per-floor if needed

7. **Compilation & Validation:**
   - [ ] `esphome compile` succeeds
   - [ ] No circular dependencies
   - [ ] Supply temp sensor shows reasonable values

### Implementation Pattern

**Update ground-floor.yaml:**
```yaml
defaults:
  safe_margin: 1.0  # °C above dew point for safety

sensor:
  - platform: combination
    id: ground_floor_max_dew_point
    name: "Max Dew Point Piano Terra"
    type: max
    unit_of_measurement: "°C"
    device_class: "temperature"
    state_class: "measurement"
    sources:
      - source: soggiorno_dew_point
      - source: cucina_dew_point
      - source: anticamera_dew_point
    on_value:
      then:
        - if:
            condition:
              # Only during cooling mode
              lambda: 'return id(summer_mode).state;'
            then:
              - climate.control:
                  id: pid_mixing_valve_ground_floor_radiant
                  target_temperature: !lambda "return (x + ${safe_margin});"
              - logger.log:
                  format: "Dew point limiting: supply temp min = %.1f°C"
                  args: ['x + ${safe_margin}']
```

## Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|------|------------|--------|------------|
| Dew point sensor failure | Medium | High | Conservative fallback (Story 13.2 documents) |
| Mode detection incorrect | Medium | Medium | Test thoroughly; heating shouldn't trigger limit |
| Safe margin too small | Low | High | Start with 1.0°C; industry standard is 2.0°C |
| Supply temp oscillation | Low | Medium | PID should handle smoothly |
