# Story 13.6: Zone Demand Aggregation Update

**Epic:** 13 - Ground Floor Hybrid Radiant-Fancoil Cooling  
**Date:** December 26, 2025  
**Status:** Complete  
**Story Points:** 2  
**Priority:** High

---

## User Story

As a **system owner enabling radiant cooling**,  
I want **the zone demand aggregation to include cooling mode activity**,  
So that **the mixing pump activates correctly during radiant cooling operations**.

---

## Story Context

**Current State:**
The `ground_floor_radiant_any_zone_open` binary sensor determines when to run the mixing pump. Currently it only checks heating PID activity.

**Problem:**
If radiant zones are cooling but the aggregation only checks heating activity, the mixing pump won't activate → no cold water flow → no cooling.

**Existing Code (ground-floor.yaml):**
```yaml
binary_sensor:
  - platform: template
    id: ground_floor_radiant_any_zone_open
    name: "Ground Floor Radiant Any Zone Open"
    device_class: running
    icon: "mdi:pump"
    lambda: |-
      return id(pid_radiant_anticamera_is_active) ||
             id(pid_radiant_bagno_terra_is_active) ||
             id(pid_radiant_cucina_is_active) ||
             id(pid_radiant_soggiorno_is_active);
```

**Required Change:**
Include cooling activity detection, or check PID action rather than just "is_active" which may be heating-specific.

## Technical Implementation

### Option A: Check PID Action (Recommended)

Update lambda to check climate action:

```yaml
binary_sensor:
  - platform: template
    id: ground_floor_radiant_any_zone_open
    name: "Ground Floor Radiant Any Zone Open"
    device_class: running
    icon: "mdi:pump"
    lambda: |-
      // Check if any radiant zone is actively heating OR cooling
      bool anticamera_active = (id(pid_radiant_anticamera).action == CLIMATE_ACTION_HEATING ||
                                id(pid_radiant_anticamera).action == CLIMATE_ACTION_COOLING);
      bool bagno_active = (id(pid_radiant_bagno_terra).action == CLIMATE_ACTION_HEATING ||
                           id(pid_radiant_bagno_terra).action == CLIMATE_ACTION_COOLING);
      bool cucina_active = (id(pid_radiant_cucina).action == CLIMATE_ACTION_HEATING ||
                            id(pid_radiant_cucina).action == CLIMATE_ACTION_COOLING);
      bool soggiorno_active = (id(pid_radiant_soggiorno).action == CLIMATE_ACTION_HEATING ||
                               id(pid_radiant_soggiorno).action == CLIMATE_ACTION_COOLING);
      
      // Also check boost mode (radiant fully open regardless of PID)
      bool boost_active = id(boost_mode_active);
      
      return anticamera_active || bagno_active || cucina_active || soggiorno_active || boost_active;
```

### Option B: Keep Existing + Add Boost

If `pid_radiant_*_is_active` already handles both modes:

```yaml
lambda: |-
  return id(pid_radiant_anticamera_is_active) ||

         id(pid_radiant_cucina_is_active) ||
         id(pid_radiant_soggiorno_is_active) ||
         id(boost_mode_active);  // Add boost mode
```

### Investigation Needed

Check `pid_sensors.yaml` to understand what `pid_radiant_*_is_active` actually measures:
- Does it use `CLIMATE_ACTION_HEATING` only?
- Or does it check any non-IDLE/OFF action?
