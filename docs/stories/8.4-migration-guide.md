### 2. HA Automation Setup (15 minutes)

Create coordinator resume automation in Home Assistant:

```yaml
automation:
  - id: epic8_coordinator_resume_soggiorno
    alias: "Epic 8: Resume Soggiorno Fancoil"
    description: "Resume fancoil PID when Epic 8 coordinator reports all conditions clear"
    trigger:
      - platform: state
        entity_id: text_sensor.soggiorno_coordinator_status
        to: "Normal (All Clear)"
        for:
          seconds: 30
    condition:
      - condition: state
        entity_id: climate.pid_fancoil_soggiorno
        state: "off"
    action:
      - service: climate.set_hvac_mode
        target:
          entity_id: climate.pid_fancoil_soggiorno
        data:
          hvac_mode: "heat"  # Or "cool" depending on season
      - service: notify.mobile_app
        data:
          title: "Epic 8: Soggiorno Climate Resumed"
          message: "Fancoil PID resumed after conditions cleared"
    mode: single
```

**Note:** Update `hvac_mode` to match current season (heat/cool/heat_cool)

## Functional Testing (Day 1: 4-6 hours)

### Test 7: Emergency Condition Trigger/Clear

**Test 7a: Emergency Trigger**

1. **Trigger emergency:**
   - Disable HA temperature sensor integration (or unplug physical sensor)
   - Start timer

2. **After 180 seconds:**
   - [ ] ESPHome log: `"Soggiorno: EMERGENCY ACTIVE - Sensor unavailable for 180 seconds"`
   - [ ] `text_sensor.soggiorno_emergency_state_sensor` → "Active"
   - [ ] `text_sensor.soggiorno_coordinator_status` → "Shutdown: Emergency (Active)"
   - [ ] ESPHome log: `"Soggiorno: SHUTDOWN TRIGGERED - Condition: Emergency (Active), Priority: 1"`
   - [ ] `climate.pid_fancoil_soggiorno` → OFF mode

**Test 7b: Emergency Clear**

1. **Clear emergency:**
   - Re-enable HA temperature sensor

   - [ ] ESPHome log: `"Soggiorno: EMERGENCY RECOVERING - Sensor restored"`
   - [ ] `text_sensor.soggiorno_emergency_state_sensor` → "Recovering"
   - [ ] `text_sensor.soggiorno_coordinator_status` → "Shutdown: Emergency (Recovering)"

   - [ ] ESPHome log: `"Soggiorno: EMERGENCY NORMAL - Recovery complete after 60 seconds"`

### Test 9: Priority Hierarchy (Emergency > Window)

**Test 9a: Simultaneous Trigger**

1. **Trigger both conditions:**
   - Disable HA temperature sensor
   - Open window
   - Wait 180 seconds

2. **After 180 seconds:**
   - [ ] Both conditions Active
   - [ ] `text_sensor.soggiorno_coordinator_status` → "Shutdown: Emergency (Active)"
   - [ ] NOT "Shutdown: Window (Active)" (Emergency has priority 1 < Window priority 2)
   - [ ] PID OFF

**Test 9b: Priority Switch**

1. **Clear higher-priority condition:**
   - Re-enable HA sensor (clears emergency)
   - Keep window open

2. **Immediately:**
   - [ ] Emergency → "Recovering"
   - [ ] Window still "Active"
   - [ ] Coordinator switches to "Shutdown: Window (Active)"
   - [ ] PID remains OFF

3. **After 60 seconds:**
   - [ ] Emergency → "Normal"
   - [ ] Window still "Active"
   - [ ] Coordinator still shows "Shutdown: Window (Active)"

**Test 9c: All Conditions Clear**

1. **Clear remaining condition:**
   - Close window

2. **Verify cascade:**
   - [ ] Window → "Recovering"
   - [ ] After 60s: Window → "Normal"
   - [ ] Coordinator → "Normal (All Clear)"
   - [ ] PID resumes via HA automation
