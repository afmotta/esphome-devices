# Story 15.2: Receive Air Quality Data on Climate Control Board

**Epic:** 15 - Air Quality Sensor Broadcasting  
**Date:** January 14, 2026  
**Status:** Done  
**Story Points:** 2  
**Priority:** High

---

## User Story

As a **climate-control board**,  
I want **to receive CO₂ and IAQ data from all first floor room sensors**,  
So that **I have real-time air quality metrics available for MEV control decisions**.

---

## Story Context

**Existing Infrastructure:**

- Climate-control board (`devices/climate-control.yaml`) already receives temp/humidity via packet_transport
- Providers already configured for first floor room sensors
- `packet_transport` receivers established pattern with `platform: packet_transport`

**Current providers section (from climate-control.yaml):**
```yaml
packet_transport:
  providers:
    - room-sensor-bagno-grande
    - room-sensor-bagno-ospiti
    - room-sensor-bagno-padronale
    - room-sensor-camera-nord
    - room-sensor-camera-sud
    - room-sensor-camera-ospiti
    - room-sensor-camera-padronale
    - room-sensor-lavanderia
```
