# Window Condition Component - Quick Reference

**Component:** `components/room_window_condition.yaml`  
**Epic:** 8 - Unified State Machine Architecture  
**Version:** 1.0

---

## Required Variables

```yaml
vars:
  zone_slug: "soggiorno"                               # Room identifier
  zone_name: "Soggiorno"                               # Display name
  ha_window_sensor_id: binary_sensor.soggiorno_window  # HA binary sensor
  pid_id: pid_soggiorno                                # PID climate entity
  window_shutdown_modes: "cooling, heating"            # CSV mode list
```

---

## Optional Variables (with defaults)

```yaml
vars:
  window_timeout: 180      # Seconds before trigger (default: 180 = 3 min)
  recovery_timeout: 60     # Seconds for recovery (default: 60 = 1 min)
  condition_id: "window"   # Global name suffix (default: "window")
  priority_value: 2        # Priority in hierarchy (default: 2)
```

---

## Exposed Interface

### Globals for Coordinator

```yaml
# State enum
${zone_slug}_window_state:
  - 0 = Normal (window closed or grace period)
  - 1 = Active (shutdown triggered)
  - 2 = Recovering (stability period)

# Priority
${zone_slug}_window_priority: 2  # Fixed value

# Debug
${zone_slug}_window_last_trigger_time: timestamp  # When last triggered
```

### Home Assistant Sensors

```yaml
text_sensor.${zone_slug}_window_state_sensor:
  - "Normal"
  - "Normal (window open Xs)"  # During grace period
  - "Active"
  - "Recovering"
```

---

## State Machine

```
Normal ──[window open + mode match + 180s]──> Active
                                                  │
                 ┌────────────────────────────────┤
                 │ [window closes]                │ [mode changes]
                 ↓                                ↓
            Recovering ──[60s]──> Normal
                 │
                 └──[window re-opens + mode match]──> Active
```

---

## Mode Configuration Examples

| Use Case          | Configuration        | Behavior                   |
| ----------------- | -------------------- | -------------------------- |
| Fancoil (typical) | `"cooling, heating"` | Shutdown in both modes     |
| Cooling only      | `"cooling"`          | Shutdown only when cooling |
| Heating only      | `"heating"`          | Shutdown only when heating |
| Disable           | `""` (empty)         | Never shutdown             |

## Coordinator Integration

**How Coordinator Uses This:**

```cpp
// Coordinator reads condition globals
int window_state = id(soggiorno_window_state);
int window_priority = id(soggiorno_window_priority);

// Apply control based on state
if (window_state == 1 || window_state == 2) {
  // Active or Recovering: Force PID OFF
  auto call = id(pid_soggiorno)->make_call();
  call.set_mode(climate::CLIMATE_MODE_OFF);
  call.perform();
}
// else: Normal, PID operates per HA state
```

## Troubleshooting

### State Stuck in Active

**Symptom:** Window closed but state remains Active  
**Check:**
- Window sensor in HA shows "off"
- ESPHome logs show interval execution
- Recovery start time global is non-zero

### No Active Trigger

**Symptom:** Window open but no shutdown  
**Check:**
- PID mode matches `window_shutdown_modes` list
- Window open duration > 180s
- ESPHome logs show mode check messages

### Text Sensor Shows "Unknown"

**Symptom:** State sensor displays "Unknown"  
**Check:**
- State global value is 0, 1, or 2
- Component included in device YAML
- Restart ESPHome device

## Testing Checklist

Quick validation after deployment:

- [ ] Open window in shutdown mode → Active after 180s
- [ ] Close window → Recovering immediately
- [ ] Wait 60s → Normal
- [ ] Open window in OFF mode → Stays Normal
- [ ] Change mode while window open → Recovering
