# Story 5.3: Room Component Migration to HA-Only Sensors

## Status
Ready for Review

## Story
**As a** room climate controller,  
**I want** all room components migrated from Modbus sensors to HA-only sensors with emergency shutdown,  
**so that** the system eliminates expensive Modbus sensor hardware while maintaining safety through emergency shutdown logic.

## Acceptance Criteria

1. All room components in `components/rooms/first_floor/*.yaml` use new `room_sensors.yaml` (v5)
2. All room components in `components/rooms/ground_floor/*.yaml` use new `room_sensors.yaml` (v5)
3. Remove `modbus_controller_address` var from all room package calls
4. Remove `ha_humidity_sensor_id` var from all room package calls (humidity not needed)
5. Each room has configurable emergency timeout (default 180s = 3 min)
6. Each room has configurable recovery stability timeout (default 60s)
7. Each room has configurable check interval (default 10s)
8. Per-room emergency state visible in Home Assistant:
   - `text_sensor.{zone}_sensor_state` shows "Normal/Emergency/Recovering"
   - `sensor.{zone}_room_sensor_last_update` shows last valid sensor timestamp
9. Test migration sequence:
   - Pilot: Migrate one room (bagno_grande), test for 24 hours
   - Verify emergency trigger (disconnect HA for >3 min, relay turns OFF, PID turns OFF)
   - Verify recovery (reconnect HA, wait 60s, system resumes)
   - After pilot success: Batch migrate all remaining rooms
10. All devices compile and validate successfully
11. Physical testing on all three devices (gruppo-miscelazione, distribuzione-piano-terra, distribuzione-primo-piano)
12. Document migration in `docs/epic-5-migration-guide.md`

## Current Architecture

**Current Room Component Call (bagno_grande.yaml):**
```yaml
packages:
  room_sensors: !include
    file: ../../room_sensors.yaml
    vars:
      zone_slug: "bagno_grande"
      zone_name: "Bagno Grande"
      modbus_controller_address: 0x0C  # ← REMOVE
      ha_temperature_sensor_id: "sensor.room_bagno_grande_temperature"
      ha_humidity_sensor_id: "sensor.room_bagno_grande_humidity"  # ← REMOVE
```

**Current `room_sensors.yaml`:**
- 3-tier failover: Modbus → HA → Emergency
- Modbus controller per room for sensor reading
- Complex lambda with multiple fallback tiers

## Proposed Architecture

**New Room Component Call (bagno_grande.yaml):**
```yaml
packages:
  room_sensors: !include
    file: ../../room_sensors.yaml
    vars:
      zone_slug: "bagno_grande"
      zone_name: "Bagno Grande"
      ha_temperature_sensor_id: "sensor.room_bagno_grande_temperature"
      # Optional overrides (use defaults if not specified):
      # emergency_timeout: 180  # seconds (3 min)
      # recovery_stability_timeout: 60  # seconds
      # check_interval: 10  # seconds
```

**New `room_sensors.yaml` (v5):**
- 2-tier logic: HA sensor → Emergency
- No Modbus controller (removed)
- State machine: Normal → Emergency → Recovering → Normal
- Configurable timeouts via vars

## Testing Scenarios

### Scenario 1: Normal Operation
- **Given:** HA connected, sensor data flowing
- **When:** Room component checks sensor state
- **Then:** `text_sensor` shows "Normal", PID operates normally

### Scenario 2: Emergency Trigger
- **Given:** HA disconnected or sensor unavailable
- **When:** 180 seconds elapse without valid sensor data
- **Then:** 
  - `text_sensor` shows "Emergency"
  - Relay turns OFF (physical output)
  - PID mode set to OFF (climate entity shows OFF)
  - Logs ERROR message with timestamp

### Scenario 3: Recovery
- **Given:** System in emergency state
- **When:** HA reconnects and provides valid sensor data for 60 seconds
- **Then:**
  - `text_sensor` shows "Recovering" for 60s
  - After 60s stability, `text_sensor` shows "Normal"
  - PID resumes operation (returns to previous mode/setpoint)
  - Relay control restored

### Scenario 4: Flapping Prevention
- **Given:** HA connection intermittent (connects/disconnects every 30s)
- **When:** Connection unstable
- **Then:**
  - System enters emergency after 3 min total unavailability
  - Recovery requires 60s continuous stability
  - Prevents rapid on/off cycling

## Rollback Plan

If critical issues discovered during pilot or batch migration:

1. **Immediate rollback:**
   - Revert room component file to previous version
   - Use `room_sensors.yaml` (old version with Modbus)
   - Reflash device
   - Restore previous vars (add back `modbus_controller_address`, `ha_humidity_sensor_id`)

2. **Keep both versions during migration:**
   - Preserve `room_sensors.yaml` as `room_sensors_v4_backup.yaml`
   - New version is `room_sensors.yaml` (v5)
   - Easy to switch back during testing
