### UDP Packet Loss Calculation (Per Room)

**Expected broadcasts (10 minutes test period):**
```
10 minutes × 60 seconds/minute ÷ 10 seconds/broadcast = 60 broadcasts
× 2 readings (temperature + humidity) = 120 packets expected per room
```

**Measurement approach:**
```bash
# Run tcpdump for 10 minutes, count packets per room
tcpdump -i wlan0 udp port 6053 -n -c 1680 > udp_capture.log
# 14 rooms × 120 packets = 1680 total packets expected

# Count packets per room
grep "soggiorno_room_temp\|soggiorno_room_humidity" udp_capture.log | wc -l
# Expected: ~120 (allow ±5 for timing variance)

# Calculate packet loss
Actual_packets = 118
Expected_packets = 120
Packet_loss = (120 - 118) / 120 × 100% = 1.67%
```

**Action thresholds:**
- **<1% loss:** Excellent, no action needed
- **1-3% loss:** Acceptable, monitor for trends
- **3-5% loss:** Marginal, investigate WiFi signal strength
- **>5% loss:** Unacceptable, relocate sensor or add WiFi extender

### Rollback Plan

**Rollback Decision Criteria (Trigger Rollback if ANY of these occur):**
1. **High packet loss:** >5% packet loss for >3 rooms
2. **Frequent sensor failures:** >10% of rooms fail over to HA tier within 24 hours
3. **Emergency shutdowns:** >2 false emergency shutdowns within 48 hours
4. **Flash/RAM critical:** >85% flash or >18% RAM (approaching limits)
5. **User-reported issues:** Room comfort complaints due to sensor issues

**Rollback Procedure (Revert to Epic 5 HA-only):**

```yaml
# Step 1: Disable UDP mode for all rooms
# locals/distribuzione-piano-terra.yaml + distribuzione-primo-piano.yaml
# Change use_udp: true → use_udp: false for all rooms

# Step 2: Recompile and flash both distribution boards
esphome compile locals/distribuzione-piano-terra.yaml
esphome upload locals/distribuzione-piano-terra.yaml

esphome compile locals/distribuzione-primo-piano.yaml
esphome upload locals/distribuzione-primo-piano.yaml

# Step 3: Verify rollback
# Check all diagnostic sensors: should show "HA" tier (Epic 5 behavior)
# PIDs should continue operating on HA sensors

# Step 4: Power off ESP32 sensors (optional, if causing network issues)
# Unplug USB cables, sensors no longer broadcasting
```

**Rollback impact:**
- System reverts to Epic 5 HA-only architecture
- UDP sensors remain deployed (can re-enable later after resolving issues)
- No loss of functionality (HA fallback works)
- Diagnostic sensors show "HA" tier instead of "UDP"

**Rollback testing:**
- Revert ground floor first (6 rooms), validate for 24 hours
- If stable, keep ground floor on Epic 5, troubleshoot first floor separately
- If issues persist: full rollback to Epic 5 for both floors
