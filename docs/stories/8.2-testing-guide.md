## Testing Overview

This guide provides manual test scenarios to validate the window condition component against the Epic 8 interface specification. All tests verify state transitions, mode-awareness logic, and interface compliance.

**Prerequisites:**
- ESPHome device with window condition component included
- Home Assistant with window binary sensor configured
- PID climate component configured in same room
- Access to ESPHome logs (via serial or network)

---

## Test Scenario 1: Normal → Active (Window Open in Shutdown Mode)

**Objective:** Verify transition to Active state when window opens for timeout duration in a shutdown mode

**Setup:**
- PID mode: `HEAT` (heating)
- `window_shutdown_modes`: `"cooling, heating"` (heating is in shutdown list)
- Window state: closed
- Initial condition state: Normal (0)

**Steps:**
1. Open window in Home Assistant (trigger binary sensor to "on")
2. Wait 180 seconds (default `window_timeout`)
3. Check ESPHome logs for state transition message
4. Check HA text sensor shows "Active"
5. Verify `${zone_slug}_window_state` global = 1
6. Verify `${zone_slug}_window_last_trigger_time` updated

**Expected Results:**
- Log message: `"${zone_name}: WINDOW ACTIVE - Open for 180 seconds in mode 1"`
- Text sensor: `"Active"`
- State global: `1`
- Priority global: `2` (unchanged)
- Last trigger time: non-zero timestamp

**Pass Criteria:** ✅ All expected results match

## Known Limitations

1. **Mode Parsing:** Simple string matching (no whitespace tolerance beyond comma-separated)
2. **Sensor Failure:** Treats unavailable sensor as "closed" (fail-safe for energy efficiency)
3. **Recovery from Mode Change:** Mode change while window open triggers recovery but coordinator may still keep PID OFF
4. **Grace Period Precision:** Text sensor shows approximate seconds due to 5s update interval

## Troubleshooting

**Issue:** State stuck in Active despite window closed
- Check: Window sensor entity in HA (verify "off" state)
- Check: ESPHome logs for interval execution
- Check: Recovery start time global (should be non-zero)

**Issue:** Window open but no Active trigger
- Check: PID current mode matches `window_shutdown_modes` list
- Check: Window open duration (must exceed 180s timeout)
- Check: ESPHome logs for mode check messages

**Issue:** Text sensor shows "Unknown"
- Check: State global value (should be 0, 1, or 2)
- Check: Component included correctly in device YAML
- Restart ESPHome device
