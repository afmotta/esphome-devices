### Remotes Configuration Strategy

**Current State:**
- `locals/` folder contains device configs with local `!include` paths
- `remotes/` folder has one config: `gruppo-miscelazione.yaml`

**Target State:**
- All three device configs in `remotes/` referencing GitHub packages
- GitHub repository URL: `github://afmotta/esphome-devices`
- Branch: `main` (or `bmad` for testing)

**Migration Pattern:**
```yaml
# Before (locals/gruppo-miscelazione.yaml)
packages:
  mixing_valve_piano_terra: !include
    file: ../components/mixing_valve.yaml
    vars: {...}

# After (remotes/gruppo-miscelazione.yaml)
packages:
  mixing_valve_piano_terra: !include
    file: github://afmotta/esphome-devices/components/mixing_valve.yaml@main
    vars: {...}
```

**Benefits:**
- OTA updates from GitHub without re-flashing
- Version control for all configurations
- Easy rollback to previous versions
- Centralized component library
