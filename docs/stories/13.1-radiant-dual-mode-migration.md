# Story 13.1: Radiant Dual-Mode Migration

**Epic:** 13 - Ground Floor Hybrid Radiant-Fancoil Cooling  
**Date:** December 26, 2025  
**Status:** Ready for Review (blocked: pre-existing MEV TemplateSwitch compilation issue)  
**Story Points:** 2  
**Priority:** High (Foundation)

---

## User Story

As a **system owner seeking improved cooling comfort**,  
I want **ground floor radiant zones to support both heating and cooling modes**,  
So that **I can use radiant cooling as the primary cooling system for superior comfort**.

---

## Story Context

**Existing System Integration:**

- **Current state:** Ground floor rooms use `heat_only_radiant.yaml` (heating only)
- **Target state:** Migrate to `radiant.yaml` (dual-mode: heat + cool)
- **Reference:** First floor already uses `radiant.yaml` for dual-mode operation
- **Touch points:**
  - `components/rooms/ground_floor/soggiorno.yaml`
  - `components/rooms/ground_floor/cucina.yaml`
  - `components/rooms/ground_floor/bagno.yaml`
  - `components/rooms/ground_floor/anticamera.yaml`

**Current Component Structure:**

```yaml
# heat_only_radiant.yaml - Current (heating only)
packages:
  pid: !include ./pid.yaml
    vars:
      circuit_slug: "radiant_${room_slug}"
      ...

climate:
  - id: !extend pid_radiant_${room_slug}
    heat_output: slow_pwm_${room_slug}

binary_sensor:
  - id: !extend summer_mode
    on_state:
      then:
        - if:
            condition:
              lambda: "return x;"
            then:
              - climate.control:
                  id: pid_radiant_${room_slug}
                  mode: "OFF"  # Currently disables radiant in summer
```

**Target Component Structure:**

```yaml
# radiant.yaml - Target (dual-mode)
packages:
  pid: !include ./heat_only_radiant.yaml

climate:
  - id: !extend pid_radiant_${room_slug}
    cool_output: slow_pwm_${room_slug}  # Adds cooling capability
```

### Summer Mode Logic Change

**Current behavior (heat_only_radiant.yaml):**
```yaml
binary_sensor:
  - id: !extend summer_mode
    on_state:
      then:
        - if:
            condition:
              lambda: "return x;"  # summer = true
            then:
              - climate.control:
                  id: pid_radiant_${room_slug}
                  mode: "OFF"  # Disable in summer
```

**Required behavior for cooling:**
```yaml
binary_sensor:
  - id: !extend summer_mode
    on_state:
      then:
        - if:
            condition:
              lambda: "return x;"  # summer = true
            then:
              - climate.control:
                  id: pid_radiant_${room_slug}
                  mode: "COOL"  # Enable cooling in summer
            else:
              - climate.control:
                  id: pid_radiant_${room_slug}
                  mode: "HEAT"  # Enable heating in winter
```
