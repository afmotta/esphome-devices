## State Machine Behavior Checklist

### State Transitions

- ✅ **Normal (0) → Active (1):** When window open + mode match + timeout exceeded (lines 208-215)
- ✅ **Active (1) → Recovering (2):** When window closes (lines 265-270)
- ✅ **Active (1) → Recovering (2):** When mode changes to non-shutdown mode (lines 240-245)
- ✅ **Recovering (2) → Normal (0):** After recovery stability period (lines 278-283)
- ✅ **Recovering (2) → Active (1):** Window re-opens during recovery (lines 218-224)
- ✅ **No direct Active → Normal:** All paths pass through Recovering state ✅

**Evidence:**
```
State Machine Implementation:

1. Normal → Active (line 208):

     id(${zone_slug}_${condition_id}_state) = 1;
     // TRANSITION: Normal (0) → Active (1)
   }

2a. Active → Recovering (window closes, line 265):
   if (current_state == 1) {
     id(${zone_slug}_${condition_id}_state) = 2;
     // TRANSITION: Active (1) → Recovering (2)
   }

2b. Active → Recovering (mode changes, line 240):
   if (current_state == 1) {
     id(${zone_slug}_${condition_id}_state) = 2;
     // TRANSITION: Active (1) → Recovering (2)
   }

3. Recovering → Normal (line 278):

     id(${zone_slug}_${condition_id}_state) = 0;

   else if (current_state == 2) {
     id(${zone_slug}_${condition_id}_state) = 1;
     // TRANSITION: Recovering (2) → Active (1)
   }
```

- ✅ **Only triggers when mode matches:** `if (mode_requires_shutdown)` check before Active transition (line 196)

**Evidence:**

    current_mode == climate::CLIMATE_MODE_HEAT) {
  mode_requires_shutdown = true;
