# Story 13.7: Diagnostic Sensors

**Epic:** 13 - Ground Floor Hybrid Radiant-Fancoil Cooling  
**Date:** December 26, 2025  
**Status:** Complete  
**Story Points:** 1  
**Priority:** Medium

---

## User Story

As a **system owner**,  
I want **clear diagnostic sensors showing the hybrid cooling system state**,  
So that **I can monitor operation and troubleshoot issues from Home Assistant**.

---

## Story Context

**Observability Goal:** Provide visibility into:
- Current cooling mode (Radiant Only vs Fancoil Boost)
- Why the system is in that mode (temperature delta)
- Dew point protection status
- Threshold being used

**Note:** Some sensors are created in Story 13.5 (coordinator). This story ensures completeness and may add supplementary diagnostics.

## Technical Implementation

### New Sensors (This Story)

**In ground-floor.yaml or coordinator component:**

```yaml
sensor:
  # Supply temperature minimum based on dew point
  - platform: template
    id: ground_floor_supply_temp_minimum
    name: "Ground Floor Supply Temp Minimum"
    unit_of_measurement: "°C"
    accuracy_decimals: 1
    device_class: temperature
    lambda: |-
      float dew_point = id(ground_floor_max_dew_point).state;
      float margin = ${safe_margin};
      return dew_point + margin;
    update_interval: 30s

  # Time in current mode (for debugging)
  - platform: template
    id: time_in_current_mode
    name: "Ground Floor Time In Current Mode"
    unit_of_measurement: "min"
    accuracy_decimals: 0
    icon: "mdi:timer"
    lambda: |-
      uint32_t elapsed_ms = millis() - id(last_mode_change_time);
      return elapsed_ms / 60000.0;  // Convert to minutes
    update_interval: 60s
```
