### Hardware Selection Recommendations

**Recommended: ESP32-C3 SuperMini + SHT31**

**Pros:**
- Total cost: ~$10-14 per sensor (lowest cost option)
- SHT31 accuracy: ±0.2°C temp, ±2% RH humidity (best accuracy)
- Compact: ESP32-C3 very small form factor
- USB-C: More modern, easier cable sourcing

**Cons:**
- Soldering required (SHT31 breakout board → ESP32 pins)
- Smaller GPIO count on ESP32-C3 (13 pins, but sufficient)

**Alternative: ESP32-WROOM DevKit + DHT22**

**Pros:**
- Pre-soldered headers (easier assembly)
- More GPIO pins (30+, overkill for this use case)
- DHT22 widely available

**Cons:**
- More expensive: ~$12-17 per sensor
- DHT22 accuracy: ±0.5°C (still acceptable for HVAC)
- Larger form factor (harder to hide)

### SHT31 Wiring Diagram

```
ESP32-C3 SuperMini       SHT31 Breakout Board
──────────────────       ────────────────────
3.3V            ───────  VCC
GND             ───────  GND
GPIO8 (SDA)     ───────  SDA
GPIO9 (SCL)     ───────  SCL
```

**ESP32-WROOM Alternative Pins:**
- SDA: GPIO21 (default)
- SCL: GPIO22 (default)

**Important:** Use 3.3V, NOT 5V! SHT31 is 3.3V logic only.

### Pilot Sensor Bench Test Procedure

**Power-On Test:**
1. Connect USB cable to pilot sensor
2. Verify blue LED blinks (ESP32 booting)
3. Wait 30 seconds for WiFi connection
4. Check router DHCP leases: sensor appears with device name

**Sensor Accuracy Test:**
1. Place calibrated thermometer next to ESP32 sensor
2. Wait 5 minutes for thermal equilibrium
3. Compare readings: difference should be <±0.5°C
4. If error >±0.5°C: apply `temp_offset` in YAML and reflash

**UDP Broadcast Test:**
1. Run tcpdump on dev machine: `tcpdump -i wlan0 udp port 6053`
2. Verify packets appear every 10 seconds
3. Example output:
   ```
   23:45:01.123 IP esp32.local.6053 > 255.255.255.255.6053: UDP, length 64
   23:45:11.234 IP esp32.local.6053 > 255.255.255.255.6053: UDP, length 64
   ```

**Long-Term Stability Test (Optional):**
- Leave pilot sensor powered for 24 hours
- Monitor uptime sensor via ESPHome web interface
- Verify no crashes/reboots (uptime should increase continuously)
- Monitor WiFi signal strength: should remain >-70 dBm

### Enclosure Options

**Option A: 3D Printed Custom Enclosure**
- **Pros:** Perfect fit, ventilation holes for accuracy, professional look
- **Cons:** Requires 3D printer access, 2 hours print time per unit
- **Files:** Search Thingiverse for "ESP32 sensor enclosure" or design custom
- **Material:** PLA or PETG, white or transparent for aesthetics

**Option B: Off-the-Shelf Project Box**
- **Pros:** Fast, no printing, durable ABS plastic
- **Cons:** Generic look, may require drilling ventilation holes
- **Recommendation:** Hammond 1551 series (~80×50×20mm), vented version
- **Cost:** $2-3 per box

**Option C: Open/Bare PCB Mounting**
- **Pros:** Zero cost, maximum ventilation (best temp accuracy)
- **Cons:** Exposed electronics, less professional, dust accumulation
- **Use Case:** Technical room, utility areas (not living spaces)

### Assembly Line Approach (Batch 13 Sensors)

**Stage 1: Soldering (2-3 hours)**
- Set up soldering station with flux, solder, helping hands
- Wire SHT31 to ESP32 (4 wires: VCC, GND, SDA, SCL)
- Use color-coded wire: red=VCC, black=GND, yellow=SDA, green=SCL
- Solder all 13 sensors before moving to next stage

**Stage 2: Testing (1-2 hours)**
- Power each sensor via USB
- Check voltage at SHT31: should be ~3.3V
- Verify sensor readings in ESPHome logs
- Label sensors that pass test

**Stage 3: Flashing (2-3 hours)**
- Create local YAML config for each room
- Flash each sensor with unique config
- Verify device name matches room
- Run UDP broadcast test for each sensor

**Stage 4: Enclosing (1-2 hours, if using enclosures)**
- Mount sensor in enclosure
- Drill ventilation holes (if not pre-vented)
- Route USB cable through cable gland
- Close enclosure, label with room name

**Total Assembly Time:** 6-10 hours for 13 sensors (plus pilot)

### Troubleshooting Common Issues

**Issue: ESP32 won't boot after soldering**
- **Cause:** Solder bridge shorting VCC to GND
- **Fix:** Check continuity between VCC and GND (should be open circuit)
- **Prevention:** Use flux, clean solder joints with isopropyl alcohol

**Issue: SHT31 sensor reads NaN (no data)**
- **Cause:** Incorrect I2C wiring (SDA/SCL swapped)
- **Fix:** Swap SDA and SCL wires
- **Prevention:** Label wires during soldering, use color codes

**Issue: WiFi won't connect**
- **Cause:** Wrong WiFi credentials in YAML
- **Fix:** Double-check `wifi_ssid` and `wifi_password` in substitutions
- **Prevention:** Use secrets file, test credentials on laptop first

**Issue: UDP broadcasts not appearing**
- **Cause:** ESP32 and dev machine on different subnets
- **Fix:** Ensure both devices on same WiFi network (same subnet)
- **Prevention:** Check router DHCP leases, verify IP addresses in same range
