## 🎯 Quick Overview

**What You're Building:**
A reusable `components/modbus_slave.yaml` component that reads climate mode and heartbeat data from the Modbus master, with timeout handling and error tracking.

**Key Outcome:**
The slave board (`distribuzione-piano-terra`) will be able to read coordination data directly from the master via Modbus, enabling autonomous operation without Home Assistant.

## 🚀 Implementation Steps

### Step 1: Create Component Skeleton (15 min)

**File:** `components/modbus_slave.yaml`

```yaml
# Component: modbus_slave.yaml
# Purpose: Read master Modbus registers with timeout handling
# Required vars:
#   - master_address: Modbus address of master (e.g., 0x01)
#   - slave_address: This slave's Modbus address (e.g., 0x02)
# Optional vars:
#   - update_interval: Polling interval (default: 10s)
#   - timeout: Read timeout (default: 30s)

defaults:
  vars:
    master_address: 0x01
    slave_address: 0x02
    update_interval: 10s
    timeout: 30s

# Check if Modbus is enabled
globals:
  - id: use_modbus_enabled
    type: bool
    restore_value: false
    initial_value: 'false'
    # Will be set from substitutions

# Modbus Controller (Slave Role)
modbus_controller:
  - id: modbus_slave
    address: ${slave_address}
    modbus_id: rs485_bus  # Must match Story 1.1 uart ID
    update_interval: never  # Slave does not initiate
```

**Test:**
```bash
esphome config locals/distribuzione-piano-terra.yaml
# Should compile without errors
```

### Step 2: Implement Climate Mode Reading (20 min)

**Add to `components/modbus_slave.yaml`:**

```yaml
# Read Climate Mode from Master (Register 200 / 0x00C8)
sensor:
  - platform: modbus_controller
    modbus_controller_id: modbus_slave
    id: modbus_climate_mode
    name: "Modbus Climate Mode"
    address: 0x00C8  # Register 200
    register_type: holding
    value_type: U_WORD
    update_interval: ${update_interval}
    filters:
      - timeout: ${timeout}  # Mark unavailable after 30s
    internal: false  # Expose to HA for monitoring
```

**Add to `devices/distribuzione-piano-terra.yaml` (packages section):**

```yaml
packages:
  # ... existing packages ...
  modbus_slave: !include
    file: ../components/modbus_slave.yaml
    vars:
      master_address: 0x01
      slave_address: 0x02
```

### Step 4: Add Error Tracking (30 min)

**Add to `components/modbus_slave.yaml`:**

```yaml
# Error Tracking Global Variable
globals:
  - id: modbus_read_error_count
    type: int
    restore_value: false
    initial_value: '0'

# Error Count Sensor
sensor:
  - platform: template
    name: "Modbus Read Error Count"
    id: modbus_error_count_sensor
    lambda: 'return id(modbus_read_error_count);'
    update_interval: 10s
    accuracy_decimals: 0

# Last Error Timestamp
text_sensor:
  - platform: template
    name: "Last Modbus Error"
    id: last_modbus_error
    update_interval: 10s

# Increment error counter on read failure
interval:
  - interval: 10s
    then:
      - lambda: |-
          if (!id(modbus_climate_mode).has_state() || 
              !id(modbus_master_heartbeat).has_state()) {
            id(modbus_read_error_count) += 1;
            id(last_modbus_error).publish_state(
              "Read failed at " + id(sntp_time).now().strftime("%Y-%m-%d %H:%M:%S")
            );
          }
```

**Note:** If `sntp_time` doesn't exist, use simple counter instead.

**Test:**

### Step 5: Slave Status Register (30 min)

**Add to `components/modbus_slave.yaml`:**

```yaml
# Slave Health Status Register (Master can poll this)
number:
  - platform: modbus_controller
    modbus_controller_id: modbus_slave
    id: slave_health_status
    name: "Slave Health Status"
    address: 0x03E8  # Register 1000
    value_type: U_WORD
    internal: true
    min_value: 0
    max_value: 3
    mode: box
    # 0=offline, 1=online, 2=degraded, 3=emergency

# Update health status based on operational state
interval:
  - interval: 10s
    then:
      - lambda: |-
          int status = 1;  // Default: healthy
          
          if (id(modbus_read_error_count) > 0) {
            status = 2;  // Degraded (errors present)
          }
          
          if (!id(modbus_climate_mode).has_state()) {
            status = 0;  // Offline (no communication)
          }
          
          id(slave_health_status).publish_state(status);
```

### Step 8: Testing (45-60 min)

#### Test 1: Climate Mode Synchronization (10 min)

1. On master: Change climate mode in Home Assistant (heat → cool)
2. Watch slave logs:
   ```bash
   esphome logs distribuzione-piano-terra.yaml | grep "Climate Mode"
   ```
3. Verify: Slave reads new mode value within 10 seconds
4. Check Home Assistant: Sensor `sensor.modbus_climate_mode` shows correct value

**Expected Result:** Master mode=2 (cool) → Slave reads register 200=2

---

#### Test 2: Heartbeat Monitoring (10 min)

1. Enable Modbus on both master and slave
2. Monitor `sensor.modbus_master_heartbeat` in Home Assistant
3. Verify: Value increments every 10 seconds
4. Expected pattern: 0 → 1 → 2 → ... → 65535 → 0 (wraps around)

**Expected Result:** Continuous heartbeat increments

---

#### Test 3: Timeout Handling (15 min)

1. Start with working Modbus communication
2. Physically disconnect RS485 cable OR disable Modbus on master
3. Watch slave logs for read failures
4. After 30 seconds: Check `sensor.modbus_climate_mode` in HA
5. Verify: Sensor shows "unavailable"
6. Verify: `sensor.modbus_read_error_count` increments
7. Verify: `sensor.last_modbus_error` shows timestamp

**Expected Result:** Graceful failure detection within 30 seconds

---

#### Test 4: Recovery from Timeout (10 min)

1. With RS485 disconnected (sensors unavailable)
2. Reconnect RS485 cable OR re-enable Modbus on master
3. Watch slave logs for successful reads
4. Verify: Sensors transition from "unavailable" → show values
5. Verify: Values match master registers

**Expected Result:** Automatic recovery without manual intervention

---

#### Test 5: Slave Status Register (10 min)

1. Check `sensor.slave_health_status` in Home Assistant
2. With healthy communication: Value should be 1 (online)
3. Introduce errors (disconnect RS485): Value should be 2 (degraded)
4. Complete communication loss: Value should be 0 (offline)

**Expected Result:** Health status accurately reflects operational state

## 🆘 Troubleshooting

### Issue: Compilation Error "modbus_slave not found"

**Solution:** Check `modbus_controller` ID matches Story 1.1 setup:
```yaml
modbus_controller:
  - id: modbus_slave  # Must match reference in sensors
```

---

### Issue: Sensors Always Show "Unavailable"

**Solution:** Verify RS485 communication:
1. Check UART wiring: TX↔RX crossed between boards
2. Check baud rate: 9600 8N1 on both boards
3. Enable debug logging: `logger: level: DEBUG`
4. Check master is exposing registers (Story 1.2)

---

### Issue: Error "has_state() not a member"

**Solution:** Update ESPHome to ≥2023.x:
```bash
pip install --upgrade esphome
```

---

### Issue: Firmware Size Too Large

**Solution:** Reduce logging or remove optional features:
```yaml
logger:
  level: INFO  # Change from DEBUG
  logs:
    modbus_controller: WARN  # Reduce verbosity
```
