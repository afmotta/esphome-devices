## Story

**As a** ground floor climate control system,  
**I want** to receive supply temperature and climate mode via UDP from the master board as primary data source (Tier 1),  
**so that** I can operate autonomously during Home Assistant outages while maintaining HA as a fallback (Tier 2).

---

## Acceptance Criteria

1. `devices/distribuzione-piano-terra.yaml` integrates UDP receiver and 3-tier failover components
2. UDP receiver listens for broadcasts from gruppo-miscelazione master board
3. 3-tier failover implements: UDP (Tier 1) → HA (Tier 2) → Emergency (Tier 3)
4. All ground floor room components use 3-tier abstracted sensors for PID control
5. Feature flag `use_udp: true` enables UDP reception
6. System continues normal operation during HA restart (UDP Tier 1 active)
7. Graceful failover from UDP to HA when UDP unavailable
8. Diagnostic sensors show active tier and failover events
9. Existing functionality unchanged (room PIDs, slow PWM outputs, pump management)
10. Rollback capability validated by setting `use_udp: false`

### 3-Tier Failover Architecture

```
┌────────────────────────────────────────┐
│  Tier 1: UDP Sensor (Primary)         │
│  - Direct from master board            │
│  - 10s update interval                 │
│  - Lowest latency                      │
└────────────┬───────────────────────────┘
             │ If unavailable (NaN or timeout)
             ▼
┌────────────────────────────────────────┐
│  Tier 2: HA Sensor (Fallback)         │
│  - homeassistant platform              │
│  - 30-60s effective update             │
│  - Requires HA connectivity            │
└────────────┬───────────────────────────┘
             │ If unavailable (NaN or timeout)
             ▼
┌────────────────────────────────────────┐
│  Tier 3: Emergency (Safety)            │
│  - Returns NaN after 180s              │
│  - Triggers coordinator shutdown       │
│  - Fail-safe behavior                  │
└────────────────────────────────────────┘
```

### Integration Pattern

Add to `devices/distribuzione-piano-terra.yaml`:
```yaml
packages:
  board: !include ../boards/a16.yaml
  wifi: !include ../boards/wifi.yaml
  
  # UDP Receiver (Epic 9)
  udp_receiver: !include
    file: ../components/udp_receiver.yaml
    vars:
      use_udp: true
      master_hostname: "gruppo-miscelazione.local"
      sensors:
        supply_temp_piano_terra: dallas_0x81000000b3e6f628
      broadcast_port: 6053
  
  # 3-Tier Failover for Supply Temp
  supply_temp_failover: !include
    file: ../components/sensor_failover_3tier.yaml
    vars:
      sensor_name: "supply_temp_piano_terra"
      tier1_udp_sensor_id: supply_temp_piano_terra_udp
      tier2_ha_sensor_id: supply_temp_piano_terra_ha
      abstracted_sensor_id: supply_temp_piano_terra_abstracted
      emergency_timeout: 180
  
  # Existing room packages (updated to use abstracted sensor)
  soggiorno: !include ../components/rooms/ground_floor/soggiorno.yaml
  # ... etc
```

### Network Configuration

- Ground floor board needs to receive UDP broadcasts from master
- Both boards must be on same network segment (no VLAN isolation)
- UDP broadcast address automatically detected or configured

### Testing During HA Restart

Critical test scenario for Epic 9 success:

1. **Initial State:** System operating normally with HA online
2. **Action:** Restart Home Assistant (e.g., `ha core restart`)
3. **Expected Behavior:**
   - UDP Tier 1 remains active (master still broadcasting)
   - Ground floor PIDs continue normal operation
   - No emergency shutdown triggered
   - System seamlessly continues without disruption
4. **After HA Restart:** System can failback to HA Tier 2 if needed, but UDP Tier 1 preferred

### Rollback Strategy

If UDP causes issues on ground floor:
1. Set `use_udp: false` in distribuzione-piano-terra.yaml
2. Recompile and flash via OTA
3. System reverts to Epic 8 2-tier failover (HA → Emergency)
4. No changes needed on master board

### Previous Epic Context

**Epic 8:** Room control coordinator with emergency/window conditions
- Ground floor rooms use room_sensors.yaml (v8) for HA temperature abstraction
- Emergency condition component detects sensor failures
- Coordinator forces PID OFF when conditions trigger

**Epic 9 Integration:**
- 3-tier failover provides supply temp abstraction (parallel to room temp abstraction)
- Emergency detection remains unchanged (room temps still from HA)
- Coordinator continues to manage PID shutdown based on conditions
