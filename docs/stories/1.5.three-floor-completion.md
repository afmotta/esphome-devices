### 0-10V Adapter Component Design

The `modbus_0_10v.yaml` component implements a **Modbus master role** to write to the 0-10V adapter (address 4):

```yaml
# Component: modbus_0_10v.yaml
# Purpose: Control 0-10V Modbus adapter for remote fancoil capacity control
# Vars: adapter_address, output_id, register_address (default 1)

modbus_controller:
  - id: modbus_0_10v_adapter
    address: ${adapter_address}
    modbus_id: rs485_bus  # Shared RS485 bus
    role: master  # Ground floor A16 acts as master to adapter

number:
  - platform: modbus_controller
    modbus_controller_id: modbus_0_10v_adapter
    id: ${output_id}_register
    name: "${friendly_name} 0-10V Output Register"
    address: ${register_address}
    value_type: U_WORD
    min_value: 0
    max_value: 10000
    mode: box
    internal: true  # Not exposed to HA, used by template output

output:
  - platform: template
    id: ${output_id}
    type: float
    write_action:
      - lambda: |-
          // Convert PID output (0.0-1.0) to adapter register (0-10000)
          uint16_t register_value = x * 10000;
          id(${output_id}_register).set_value(register_value);
```

**Design Notes:**
- Ground floor A16 acts as **Modbus master** to the 0-10V adapter
- Master A6 acts as **Modbus master** to both A16 slaves
- Both master roles operate on the **same shared RS485 bus**
- Addressing prevents conflicts (master=1, slaves=2/3, adapter=4)
- Template output converts PID float (0-1.0) to register integer (0-10000)

### RS485 Bus Topology and Termination

**Physical Wiring:**
```
Master A6 (addr 1) ───RS485───┬─── Ground Floor A16 (addr 2)
     [Term]                   │
                              └─── First Floor A16 (addr 3)
                                        [Term]
                                   
Ground Floor A16 also connects to 0-10V Adapter (addr 4) via same bus
```

**Termination Requirements:**
- **120Ω resistor** at Master A6 (physical start of bus)
- **120Ω resistor** at First Floor A16 (physical end of bus)
- 0-10V adapter does NOT need termination (not at physical end)

**Bus Characteristics:**
- Baud rate: 9600 baud 8N1 (no parity)
- Maximum cable length: 1200m (typical residential: <50m)
- Twisted-pair shielded cable required
- All devices share same A/B signal lines

### Critical Success Factors

1. **No Addressing Conflicts**: Unique addresses (1, 2, 3, 4) prevent bus collisions
2. **Bus Termination**: Proper 120Ω termination at both physical ends critical for reliable communication
3. **Polling Cycle Timing**: Master must complete polling both slaves within 500ms window
4. **Gradual Rollout**: Deploy with `use_modbus: false` first, enable after validation
5. **Component Reuse**: First floor reuses existing components - no new component development needed except modbus_0_10v.yaml
6. **0-10V Scaling**: Correct scaling (0-10000 register = 0-10V) critical for fancoil control
7. **Shared Bus**: Ground floor A16 acts as both slave (to master) and master (to adapter) on same bus
