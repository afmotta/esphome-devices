## Story

**As a** system architect,  
**I want** a reusable UDP sender component that broadcasts sensor data from the master board,  
**so that** slave boards can receive critical sensor values (supply temperatures, climate mode) directly over the network without requiring Home Assistant as an intermediary.

---

## Acceptance Criteria

1. Component package `components/udp_sender.yaml` is created following ESPHome package composition pattern
2. Component accepts configurable variables: `sensors` (list of sensor IDs), `text_sensors` (list of text sensor IDs), `update_interval` (seconds), `broadcast_port` (UDP port number)
3. Component broadcasts JSON-formatted UDP packets containing sensor name, value, unit, and timestamp
4. Broadcasts are sent to network broadcast address (e.g., 192.168.1.255) on specified port (default 6053)
5. Component includes feature flag support via `use_udp` substitution variable (default: false)
6. When `use_udp: false`, component does nothing (safe no-op for rollback)
7. Component includes diagnostic counter sensor tracking packets sent
8. JSON payload format is documented in component header comments
9. Component successfully compiles on KC868-A6 board (gruppo-miscelazione) without firmware size overflow
10. UDP packets are visible via network capture tools (Wireshark/tcpdump) during testing

### ESPHome UDP Implementation

ESPHome doesn't have a built-in `udp` platform. Implementation approaches:

**Option 1: Native ESP-IDF UDP Socket (Recommended)**
```yaml
lambda: |-
  #include <sys/socket.h>
  #include <netinet/in.h>
  
  int sock = socket(AF_INET, SOCK_DGRAM, IPPROTO_UDP);
  struct sockaddr_in dest_addr;
  dest_addr.sin_family = AF_INET;
  dest_addr.sin_port = htons(${broadcast_port});
  inet_pton(AF_INET, "192.168.1.255", &dest_addr.sin_addr);
  
  std::string payload = build_json_payload();
  sendto(sock, payload.c_str(), payload.length(), 0, 
         (struct sockaddr*)&dest_addr, sizeof(dest_addr));
  close(sock);
```

**Option 2: WiFiUDP Library (ESP-IDF framework)**
```yaml
lambda: |-
  WiFiUDP udp;
  udp.beginPacket("192.168.1.255", ${broadcast_port});
  udp.print(build_json_payload().c_str());
  udp.endPacket();
```

### JSON Serialization Strategy

Manual string building for MVP (minimal firmware size):
```cpp
std::string json = "{"timestamp":" + std::to_string(id(sntp_time).now().timestamp);
json += ","sensors":[";
bool first = true;
// Loop through sensors
if (!isnan(id(sensor1).state)) {
  if (!first) json += ",";
  json += "{"id":"sensor1","value":" + std::to_string(id(sensor1).state) + "}";
  first = false;
}
json += "]}";
```
