## Story

**As a** system architect,  
**I want** the master board to broadcast sensor data using ESPHome's native packet_transport platform,  
**so that** slave boards can receive critical sensor values (supply temperatures) directly over the network without requiring Home Assistant as an intermediary.

---

## Acceptance Criteria

1. Master board device file (`devices/gruppo-miscelazione.yaml`) implements ESPHome `packet_transport` platform with `udp` configuration
2. Configuration broadcasts both Dallas temperature sensors (ground floor + first floor mixing outputs)
3. Each sensor is mapped to a `broadcast_id` for receiver identification
4. UDP configuration uses standard port and update interval (10s default)
5. Configuration is inline in device file (no separate component package needed)
6. Firmware successfully compiles on KC868-A6 board (gruppo-miscelazione) without size overflow
7. UDP packets are visible via network capture tools (Wireshark/tcpdump) during testing
8. Existing functionality (PID controllers, mixing valves, HA integration) remains unchanged
9. Broadcast IDs follow naming convention: `mixing_output_{floor}_temperature`
10. Implementation can be easily disabled by commenting out `udp` and `packet_transport` sections

### ESPHome Packet Transport Platform

ESPHome provides native `packet_transport` platform for UDP sensor broadcasting:

**Configuration Pattern:**
```yaml
udp:
  id: udp_packet_transport

packet_transport:
  - platform: udp
    udp_id: udp_packet_transport
    update_interval: 10s
    sensors:
      - id: sensor_id_on_this_board
        broadcast_id: logical_name_for_receivers
```

**Key Features:**
- Native ESPHome implementation (no custom code needed)
- Automatic JSON serialization
- Built-in error handling
- Minimal firmware overhead
- Direct sensor ID mapping

### Testing Standards

**Compile Test:**
```bash
esphome compile devices/gruppo-miscelazione.yaml
```

**Network Capture Test:**
```bash
sudo tcpdump -i eth0 -n udp -A
# Or with Wireshark: filter by 'udp'
```

**Implementation Example:**
```yaml
# In devices/gruppo-miscelazione.yaml
udp:
  id: udp_packet_transport

packet_transport:
  - platform: udp
    udp_id: udp_packet_transport
    update_interval: 10s
    sensors:
      - id: dallas_0x81000000b3e6f628
        broadcast_id: mixing_output_ground_floor_temperature
      - id: dallas_0xe9000000b366ed28
        broadcast_id: mixing_output_first_floor_temperature
```
