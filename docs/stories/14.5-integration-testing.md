# Story 14.5: Integration Testing

**Epic:** 14 - Per-Room Fancoil Boost Control  
**Date:** December 26, 2025  
**Status:** Ready  
**Story Points:** 2  
**Priority:** High

---

## User Story

As a **system owner**,  
I want **comprehensive testing of the per-room boost control**,  
So that **I can validate independent room operation before production use**.

---

## Acceptance Criteria

### Test Scenarios

1. **Scenario A: Soggiorno Hot, Cucina Cool**
   - [ ] Soggiorno delta = 5°C, Cucina delta = 1°C
   - [ ] Only Soggiorno enters boost
   - [ ] Only slow_pwm_soggiorno at 100%
   - [ ] Only pid_fancoil_soggiorno in COOL mode
   - [ ] Cucina remains in radiant-only mode

2. **Scenario B: Both Rooms Hot**
   - [ ] Both deltas > threshold
   - [ ] Both rooms enter boost independently
   - [ ] Both fancoils active
   - [ ] Both radiant at 100%

3. **Scenario C: Soggiorno Cools Down**
   - [ ] Soggiorno was in boost, delta drops below hysteresis
   - [ ] Soggiorno exits boost after 10 minutes
   - [ ] Cucina remains in its current state
   - [ ] Soggiorno radiant PID resumes control

4. **Scenario D: Independent Timing**
   - [ ] Soggiorno enters boost at T=0
   - [ ] Cucina enters boost at T=5min
   - [ ] Soggiorno can exit at T=10min
   - [ ] Cucina must wait until T=15min to exit

5. **Scenario E: Winter Mode**
   - [ ] Both rooms in boost
   - [ ] summer_mode changes to false
   - [ ] Both rooms exit boost
   - [ ] Both fancoils OFF

### Validation Checks

6. **Diagnostic Sensors:**
   - [ ] `soggiorno_cooling_mode` shows correct state
   - [ ] `cucina_cooling_mode` shows correct state
   - [ ] Per-room deltas visible and accurate
   - [ ] Per-room time-in-mode incrementing

7. **Zone Demand:**
   - [ ] Fancoil pump runs when Soggiorno in boost (only)
   - [ ] Fancoil pump runs when Cucina in boost (only)
   - [ ] Fancoil pump runs when both in boost
   - [ ] Fancoil pump off when neither in boost

8. **Resource Usage:**
   - [ ] Flash usage < 80%
   - [ ] RAM usage reasonable
   - [ ] No memory leaks
