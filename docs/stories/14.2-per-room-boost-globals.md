# Story 14.2: Per-Room Boost Globals and State

**Epic:** 14 - Per-Room Fancoil Boost Control  
**Date:** December 26, 2025  
**Status:** Ready  
**Story Points:** 2  
**Priority:** High

---

## User Story

As a **system owner with independent room control**,  
I want **separate boost state tracking for each room**,  
So that **each room can enter/exit boost mode independently**.

---

## Story Context

**Current State:** Single shared globals:
- `boost_mode_active` (bool) - shared for both rooms
- `last_mode_change_time` (uint32_t) - shared for both rooms

**Target State:** Per-room globals:
- `soggiorno_boost_active` / `cucina_boost_active`
- `soggiorno_last_mode_change` / `cucina_last_mode_change`

---

## Acceptance Criteria

### Functional Requirements

1. **Soggiorno Boost State:**
   - [ ] Create `soggiorno_boost_active` global (bool, restore_value: true)
   - [ ] Create `soggiorno_last_mode_change` global (uint32_t)
   - [ ] Independent from Cucina state

2. **Cucina Boost State:**
   - [ ] Create `cucina_boost_active` global (bool, restore_value: true)
   - [ ] Create `cucina_last_mode_change` global (uint32_t)
   - [ ] Independent from Soggiorno state

3. **Deprecation:**
   - [ ] Remove `boost_mode_active` global
   - [ ] Remove `last_mode_change_time` global

4. **State Persistence:**
   - [ ] Both boost states survive reboot (restore_value: true)
   - [ ] Mode change times reset to 0 on boot (no restore)

---

## Technical Implementation

### Replace globals in coordinator

```yaml
globals:
  # Soggiorno boost state
  - id: soggiorno_boost_active
    type: bool
    initial_value: "false"
    restore_value: true
  - id: soggiorno_last_mode_change
    type: uint32_t
    initial_value: "0"

  # Cucina boost state  
  - id: cucina_boost_active
    type: bool
    initial_value: "false"
    restore_value: true
  - id: cucina_last_mode_change
    type: uint32_t
    initial_value: "0"
```
