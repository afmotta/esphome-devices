## Acceptance Criteria

### Pre-Migration Requirements

1. **Baseline Capture:**
   - Document current Soggiorno behavior (48h of sensor data)
   - Record current entity names (for HA automation updates)
   - Capture baseline temperature control quality (±1°C variance)
   - Git commit current working configuration with tag `epic-7-soggiorno-baseline`

2. **Rollback Preparation:**
   - Test rollback procedure on non-production room first
   - Document rollback steps in migration strategy
   - Confirm rollback time < 10 minutes (compile + upload)
   - Have Epic 7 configuration readily accessible via Git tag

3. **HA Automation Setup:**
   - Create coordinator resume automation (see migration strategy doc)
   - Test automation logic with simulated sensor states
   - Verify automation triggers within 30s of conditions clearing

### Migration Execution

4. **Package Configuration Update:**
   - Update `components/rooms/first_floor/soggiorno.yaml`:
     - Remove Epic 5 `room_sensors.yaml` package
     - Remove Epic 5 `room_emergency_shutdown.yaml` package
     - Remove Epic 7 `room_window_detection.yaml` package
     - Add Epic 8 `room_emergency_condition.yaml` package
     - Add Epic 8 `room_window_condition.yaml` package
     - Add Epic 8 `room_control_coordinator.yaml` package
   - Pass correct vars to each component (zone_slug, zone_name, etc.)

5. **Configuration Validation:**
   - Run `esphome config devices/distribuzione-primo-piano.yaml`
   - Verify no YAML syntax errors
   - Verify no missing global references
   - Verify no duplicate entity IDs
   - Check expected entity changes:
     - New: `text_sensor.soggiorno_emergency_state`
     - New: `text_sensor.soggiorno_window_state`
     - New: `text_sensor.soggiorno_coordinator_status`
     - Removed: `text_sensor.soggiorno_sensor_state` (Epic 5)
     - Removed: `text_sensor.soggiorno_window_detection_state` (Epic 7)

6. **Deployment:**
   - Compile: `esphome compile devices/distribuzione-primo-piano.yaml`
   - Upload: `esphome upload devices/distribuzione-primo-piano.yaml`
   - Monitor boot logs for errors via `esphome logs`
   - Verify all conditions initialize to Normal (state=0)
   - Verify coordinator shows "Normal (All Clear)"

### Functional Testing (Day 1: 4-6 hours)

7. **Emergency Condition Testing:**
   - **Test 7a:** Trigger emergency (disconnect HA sensor)
     - After 180s: Verify `soggiorno_emergency_state` → Active (1)
     - Verify `soggiorno_coordinator_status` → "Shutdown: Emergency (Active)"
     - Verify PID forced to OFF mode
     - Verify ESPHome logs: "Emergency ACTIVE (sensor unavailable 180s)"
   - **Test 7b:** Clear emergency (reconnect sensor)
     - Immediately: Verify `soggiorno_emergency_state` → Recovering (2)
     - Verify coordinator → "Shutdown: Emergency (Recovering)"
     - Verify PID remains OFF
     - After 60s: Verify state → Normal (0)
     - Verify coordinator → "Normal (All Clear)"
     - Verify HA automation resumes PID (within 30s)

8. **Window Condition Testing:**
   - **Test 8a:** Trigger window (open physical window or simulate binary_sensor)
     - After 180s: Verify `soggiorno_window_state` → Active (1)
     - Verify coordinator → "Shutdown: Window (Active)"
     - Verify PID forced to OFF mode
     - Verify logs: "Window ACTIVE"
   - **Test 8b:** Clear window (close window)
     - Immediately: Verify state → Recovering (2)
     - After 60s: Verify state → Normal (0)
     - Verify PID resumes via HA automation

9. **Priority Hierarchy Testing:**
   - **Test 9a:** Simultaneous trigger (emergency + window both active)
     - Trigger both conditions
     - After 180s: Verify both states Active (1)
     - Verify coordinator shows "Shutdown: Emergency (Active)" (priority 1 < 2)
     - Verify PID OFF
   - **Test 9b:** Priority switch (clear emergency, window still active)
     - Clear emergency condition
     - Verify emergency → Recovering (2), window still Active (1)
     - Verify coordinator switches to "Shutdown: Window (Active)"
     - After 60s: Verify emergency → Normal (0)
     - Verify coordinator still shows "Shutdown: Window (Active)"
   - **Test 9c:** All conditions clear
     - Clear window condition
     - Verify window → Recovering (2)
     - After 60s: Verify window → Normal (0)
     - Verify coordinator → "Normal (All Clear)"
     - Verify PID resumes

10. **Mode-Aware Window Testing:**
    - **Test 10a:** Window in heating mode (should trigger if configured)
      - Set PID to heating mode
      - Open window
      - Verify window condition behavior per `window_shutdown_modes` var
    - **Test 10b:** Window mode awareness validation
      - Verify window only triggers for configured modes
      - Verify correct mode-aware logic from Epic 7 preserved

### Soak Testing (Days 2-4: 48-72 hours)

11. **Stability Validation:**
    - Run for 48-72 hours in normal operation
    - Monitor ESPHome logs for errors/warnings
    - Track any unexpected state transitions
    - Verify no memory leaks (check ESP32 free heap)
    - Confirm coordinator interval runs every 5s without delays

12. **Climate Control Quality:**
    - Compare temperature variance vs. baseline (target: ±1°C)
    - Verify setpoint tracking during normal operation
    - Confirm no temperature overshoot/undershoot
    - Check PID resume behavior matches Epic 7 quality

13. **Observability Validation:**
    - Verify HA dashboard shows all Epic 8 sensors correctly
    - Confirm diagnostic text sensors update in real-time
    - Test HA history graphs for condition state sensors
    - Validate coordinator status sensor provides clear information

### Regression Testing

14. **Existing Functionality Preserved:**
    - Verify room temperature control operates normally
    - Confirm no impact on other rooms (epic-7 still operational)
    - Check no HA automation breakage (update automations as needed)
    - Validate no performance degradation (ESP32 CPU/memory)

15. **Edge Case Validation:**
    - Test rapid condition trigger/clear cycles (flapping detection)
    - Test condition trigger during ESP32 boot
    - Test HA unavailable scenario (conditions should handle gracefully)
    - Test simultaneous multi-room operations (no cross-talk)

**Mitigation:**
- Extensive functional testing before soak test
- Real-time log monitoring during deployment
- Immediate rollback capability (< 10 min)

**Mitigation:**
