## Story
**As a** room climate controller,  
**I want** a simplified `room_sensors.yaml` component that uses only HA sensors with emergency shutdown,  
**so that** expensive Modbus sensors are eliminated while maintaining safety through 3-minute emergency timeout.

## Acceptance Criteria

1. Create new `components/room_sensors.yaml` (v5) replacing the old 3-tier Modbus version
2. Remove all Modbus sensor platforms and controller definitions
3. Implement 2-tier logic: HA sensor → Emergency (no Modbus tier)
4. Add configurable timeout parameters via vars:
   - `emergency_timeout` (default: 180s = 3 minutes)
   - `recovery_stability_timeout` (default: 60s)
   - `check_interval` (default: 10s)
5. Implement state machine: Normal → Emergency → Recovering → Normal
6. Repurpose `text_sensor.{zone}_temp_source` to `text_sensor.{zone}_sensor_state` showing "Normal/Emergency/Recovering"
7. Remove humidity sensor support (not needed for PID control)
8. Preserve diagnostic sensor: `sensor.{zone}_room_sensor_last_update`
9. Component validates and compiles without errors
10. Code complexity reduced by ~60% (221 lines → ~100 lines)
11. Comprehensive inline documentation and comments

## Proposed Architecture

**New `room_sensors.yaml` (v5 - ~100 lines):**
```yaml
# Component: room_sensors.yaml (v5)
# Purpose: HA-only room temperature sensing with emergency shutdown
# Epic: Epic 5 - HA-Only Sensors with Emergency Shutdown
#
# Required vars:
#   - zone_slug: Room identifier (e.g., "bagno_grande")
#   - zone_name: Display name (e.g., "Bagno Grande")
#   - ha_temperature_sensor_id: HA entity ID (e.g., "sensor.room_bagno_grande_temperature")
#
# Optional vars:
#   - emergency_timeout: Seconds before emergency (default: 180 = 3 min)
#   - recovery_stability_timeout: Seconds of stability for recovery (default: 60)
#   - check_interval: Sensor check frequency (default: 10s)
#
# Exposes:
#   - sensor.{zone}_room_temp_abstracted: Temperature for PID (internal)
#   - text_sensor.{zone}_sensor_state: "Normal" / "Emergency" / "Recovering"
#   - sensor.{zone}_room_sensor_last_update: Last valid sensor timestamp
#
# State Machine:
#   Normal → Emergency (after emergency_timeout with no valid sensor)
#   Emergency → Recovering (HA sensor valid)
#   Recovering → Normal (after recovery_stability_timeout)
#   Recovering → Emergency (if sensor fails again)

defaults:
  emergency_timeout: 180  # 3 minutes
  recovery_stability_timeout: 60  # 1 minute
  check_interval: 10  # 10 seconds

globals:
  - id: ${zone_slug}_last_valid_time
    type: unsigned long
    restore_value: false
    initial_value: "0"
    
  - id: ${zone_slug}_emergency_triggered
    type: bool
    restore_value: false
    initial_value: "false"
    
  - id: ${zone_slug}_recovery_start_time
    type: unsigned long
    restore_value: false
    initial_value: "0"

# ============================================================================
# HOME ASSISTANT SENSOR (Primary and Only Source)
# ============================================================================

sensor:
  - platform: homeassistant
    entity_id: ${ha_temperature_sensor_id}
    id: ${zone_slug}_room_temperature_ha
    internal: true
    on_value:
      then:
        - lambda: |-
            // Valid sensor update received
            id(${zone_slug}_last_valid_time) = millis();
            ESP_LOGD("room_sensors", "${zone_name}: Valid sensor update received");

  # ============================================================================
  # ABSTRACTED TEMPERATURE WITH EMERGENCY LOGIC
  # ============================================================================

  - platform: template
    id: ${zone_slug}_room_temp_abstracted
    name: "${zone_name} Room Temperature"
    device_class: temperature
    state_class: measurement
    unit_of_measurement: "°C"
    accuracy_decimals: 1
    update_interval: ${check_interval}
    internal: true  # Used by PID, not exposed to HA
    lambda: |-
      unsigned long now = millis();
      unsigned long time_since_valid = (now - id(${zone_slug}_last_valid_time)) / 1000;  // seconds
      
      // Check if HA sensor has valid data
      if (id(${zone_slug}_room_temperature_ha).has_state()) {
        float ha_temp = id(${zone_slug}_room_temperature_ha).state;
        
        if (!isnan(ha_temp)) {
          // NORMAL STATE: Valid HA sensor data
          
          if (id(${zone_slug}_emergency_triggered)) {
            // Transitioning from Emergency to Recovering
            if (id(${zone_slug}_recovery_start_time) == 0) {
              id(${zone_slug}_recovery_start_time) = now;
              ESP_LOGW("room_sensors", "${zone_name}: RECOVERING - Sensor back online");
            }
            
            // Check if recovery stability period completed
            unsigned long recovery_duration = (now - id(${zone_slug}_recovery_start_time)) / 1000;
            if (recovery_duration >= ${recovery_stability_timeout}) {
              // Recovery complete, return to normal
              id(${zone_slug}_emergency_triggered) = false;
              id(${zone_slug}_recovery_start_time) = 0;
              ESP_LOGI("room_sensors", "${zone_name}: NORMAL - Recovery complete after %lu seconds", recovery_duration);
            } else {
              ESP_LOGD("room_sensors", "${zone_name}: RECOVERING - %lu / %d seconds", recovery_duration, ${recovery_stability_timeout});
            }
          }
          
          return ha_temp;
        }
      }
      
      // NO VALID SENSOR DATA
      
      // Reset recovery if we were in recovery state
      if (id(${zone_slug}_emergency_triggered) && id(${zone_slug}_recovery_start_time) > 0) {
        ESP_LOGW("room_sensors", "${zone_name}: RECOVERY FAILED - Sensor lost again");
        id(${zone_slug}_recovery_start_time) = 0;
      }
      
      // Check if emergency timeout exceeded
      if (time_since_valid >= ${emergency_timeout}) {
        if (!id(${zone_slug}_emergency_triggered)) {
          // TRIGGER EMERGENCY
          id(${zone_slug}_emergency_triggered) = true;
          ESP_LOGE("room_sensors", "${zone_name}: EMERGENCY TRIGGERED - No valid sensor for %lu seconds", time_since_valid);
          
          // Emergency actions handled in on_value_range below
        } else {
          ESP_LOGW("room_sensors", "${zone_name}: EMERGENCY - Still no sensor (%lu seconds)", time_since_valid);
        }
      } else {
        // Pre-emergency warning
        ESP_LOGW("room_sensors", "${zone_name}: WARNING - No sensor data for %lu / %d seconds", time_since_valid, ${emergency_timeout});
      }
      
      // Return NaN to indicate no valid data
      return NAN;
    
    # Emergency action: Sensor returns NaN
    on_value_range:
      - below: -100.0  # Will never match valid temperature
        then:
          - lambda: |-
              if (id(${zone_slug}_emergency_triggered)) {
                // Emergency actions (triggers once when emergency state entered)
                // Note: Actual relay/PID control done at room component level
                ESP_LOGE("room_sensors", "${zone_name}: EMERGENCY ACTIVE");
              }

  # ============================================================================
  # DIAGNOSTIC SENSOR: Last Update Timestamp
  # ============================================================================

  - platform: template
    id: ${zone_slug}_room_sensor_last_update
    name: "${zone_name} Sensor Last Update"
    device_class: timestamp
    update_interval: 30s
    lambda: |-
      if (id(${zone_slug}_last_valid_time) == 0) {
        return {};  // No data yet
      }
      unsigned long seconds_since = (millis() - id(${zone_slug}_last_valid_time)) / 1000;
      // Return seconds ago as timestamp
      return seconds_since;

# ============================================================================
# STATE SENSOR: Visual indicator of system state
# ============================================================================

text_sensor:
  - platform: template
    id: ${zone_slug}_sensor_state
    name: "${zone_name} Sensor State"
    icon: "mdi:state-machine"
    update_interval: 5s
    lambda: |-
      if (id(${zone_slug}_emergency_triggered)) {
        if (id(${zone_slug}_recovery_start_time) > 0) {
          return {"Recovering"};
        } else {
          return {"Emergency"};
        }
      } else {
        return {"Normal"};
      }
```

## State Machine Logic

```
┌──────────────────────────────────────────────────────────────┐
│                      INITIAL STATE                           │
│                         Normal                                │
│  - HA sensor providing valid data                            │
│  - PID operating normally                                    │
│  - Relay control active                                      │
└──────────────────────────────────────────────────────────────┘
                               │
                               │ HA sensor unavailable
                               │ for > emergency_timeout (180s)
                               ▼
┌──────────────────────────────────────────────────────────────┐
│                       EMERGENCY STATE                         │
│  - emergency_triggered = true                                │
│  - Sensor returns NAN                                        │
│  - Relay turned OFF (by room component)                      │
│  - PID mode set to OFF (by room component)                   │
│  - Logs ERROR messages                                       │
└──────────────────────────────────────────────────────────────┘
                               │
                               │ HA sensor valid again
                               ▼
┌──────────────────────────────────────────────────────────────┐
│                     RECOVERING STATE                          │
│  - recovery_start_time set                                   │
│  - Sensor returns valid temperature                          │
│  - Still in emergency mode (relay OFF, PID OFF)              │
│  - Waiting for stability                                     │
└──────────────────────────────────────────────────────────────┘
                               │
                    ┌──────────┴──────────┐
                    │                     │
      Sensor fails again       Stable for recovery_stability_timeout (60s)
                    │                     │
                    ▼                     ▼
            Back to EMERGENCY      Return to NORMAL
            (recovery_start_time   (emergency_triggered = false)
             reset to 0)           (System resumes operation)
```

## Variable Configuration Reference

### Required Variables
- **`zone_slug`**: Lowercase identifier for IDs (e.g., `"bagno_grande"`)
- **`zone_name`**: Human-readable name (e.g., `"Bagno Grande"`)
- **`ha_temperature_sensor_id`**: HA entity ID (e.g., `"sensor.room_bagno_grande_temperature"`)

### Optional Variables (with defaults)
- **`emergency_timeout`**: Seconds before emergency (default: `180`)
  - Use case: Faster emergency for critical zones, slower for less critical
- **`recovery_stability_timeout`**: Seconds of stability required (default: `60`)
  - Use case: Longer for flaky networks, shorter for stable networks
- **`check_interval`**: How often to check sensor state (default: `10`)
  - Use case: Faster for responsive control, slower to reduce CPU load

### Removed Variables (from v4)
- ~~`modbus_controller_address`~~ - No longer needed (Modbus removed)
- ~~`ha_humidity_sensor_id`~~ - No longer needed (humidity not used)

## Testing Strategy

### Unit Testing (per room)
1. **Normal Operation Test:**
   - HA sensor provides valid temperature
   - `text_sensor` shows "Normal"
   - PID receives valid temperature value

2. **Emergency Trigger Test:**
   - Disconnect HA or make sensor unavailable
   - Wait 180+ seconds
   - Verify `text_sensor` shows "Emergency"
   - Verify sensor returns NAN
   - Verify logs show ERROR message

3. **Recovery Test:**
   - Start in Emergency state
   - Reconnect HA sensor
   - Verify `text_sensor` shows "Recovering"
   - Wait 60+ seconds
   - Verify `text_sensor` shows "Normal"
   - Verify sensor returns valid temperature

4. **Flapping Prevention Test:**
   - HA sensor connects/disconnects repeatedly (every 30s)
   - Verify emergency not triggered until 180s total unavailability
   - Verify recovery requires 60s continuous stability

### Integration Testing (multiple rooms)
1. All rooms in Normal state simultaneously
2. One room emergency doesn't affect others
3. Multiple rooms in emergency simultaneously
4. Staggered recovery (rooms recover at different times)
