# Story 16.2: Air Quality → Fan Speed Mapping

**Epic:** 16 - First Floor MEV Intelligent Control  
**Date:** January 15, 2026  
**Status:** Done  
**Story Points:** 2  
**Priority:** High

---

## User Story

As a **MEV control system**,  
I want **automatic fan speed adjustment based on CO₂ and IAQ levels**,  
So that **air quality is maintained without wasting energy when conditions are good**.

---

## Story Context

**Inputs (from Epic 15):**

- `sensor.first_floor_max_co2` — Highest CO₂ reading across 8 rooms (ppm)
- `sensor.first_floor_max_iaq` — Highest IAQ index across 8 rooms (0-500+)

**Design Decision (from Brainstorming):**

- "Max demand wins" — highest demand from either CO₂ or IAQ sets fan speed
- Baseline fan speed 20% (energy-efficient when air quality is good)
- Maximum fan speed 90% (preserve some headroom)
- 5 demand tiers for smooth progression

---

## Acceptance Criteria

### Functional Requirements

1. **CO₂ → Demand Mapping:**
   - [x] CO₂ <600 ppm → 20% demand (excellent)
   - [x] CO₂ 600-800 ppm → 35% demand (good)
   - [x] CO₂ 800-1000 ppm → 50% demand (moderate)
   - [x] CO₂ 1000-1400 ppm → 70% demand (poor)
   - [x] CO₂ >1400 ppm → 90% demand (bad)

2. **IAQ → Demand Mapping:**
   - [x] IAQ <100 → 20% demand (excellent)
   - [x] IAQ 100-150 → 35% demand (good)
   - [x] IAQ 150-250 → 50% demand (moderate)
   - [x] IAQ 250-350 → 70% demand (poor)
   - [x] IAQ >350 → 90% demand (bad)

3. **Combined Demand:**
   - [x] Final AQ demand = max(CO₂ demand, IAQ demand)
   - [x] Exposed as `sensor.mev_air_quality_demand`

4. **Fallback Behavior:**
   - [x] If CO₂ unavailable → treat as 600 ppm (20% demand)
   - [x] If IAQ unavailable → treat as 100 (20% demand)

### Technical Requirements

5. **HA Template Sensor:**
   - [x] Create `sensor.mev_air_quality_demand` template sensor
   - [x] Updates when either input changes
   - [x] Unit: %

6. **Integration with Cascade:**
   - [x] AQ demand is baseline for all humidity stages
   - [x] Stage 1 (Elevated) adds +20% to AQ demand
   - [x] Stages 2-3 set minimums (70%, 90%)

### Implementation

**HA Template Sensor:**
```yaml
template:
  - sensor:
      - name: "MEV Air Quality Demand"
        unique_id: mev_air_quality_demand
        unit_of_measurement: "%"
        device_class: power_factor
        icon: mdi:fan
        state: >
          {% set co2 = states('sensor.first_floor_max_co2') | float(600) %}
          {% set iaq = states('sensor.first_floor_max_iaq') | float(100) %}
          
          {# CO₂ → Demand mapping #}
          {% set co2_demand = 
            90 if co2 > 1400 else
            70 if co2 > 1000 else
            50 if co2 > 800 else
            35 if co2 > 600 else 20 %}
          
          {# IAQ → Demand mapping #}
          {% set iaq_demand = 
            90 if iaq > 350 else
            70 if iaq > 250 else
            50 if iaq > 150 else
            35 if iaq > 100 else 20 %}
          
          {{ [co2_demand, iaq_demand] | max }}
        
        attributes:
          co2_level: "{{ states('sensor.first_floor_max_co2') | float(600) | round(0) }}"
          co2_demand: >
            {% set co2 = states('sensor.first_floor_max_co2') | float(600) %}
            {{ 90 if co2 > 1400 else 70 if co2 > 1000 else 50 if co2 > 800 else 35 if co2 > 600 else 20 }}
          iaq_level: "{{ states('sensor.first_floor_max_iaq') | float(100) | round(0) }}"
          iaq_demand: >
            {% set iaq = states('sensor.first_floor_max_iaq') | float(100) %}
            {{ 90 if iaq > 350 else 70 if iaq > 250 else 50 if iaq > 150 else 35 if iaq > 100 else 20 }}
          driving_signal: >
            {% set co2 = states('sensor.first_floor_max_co2') | float(600) %}
            {% set iaq = states('sensor.first_floor_max_iaq') | float(100) %}
            {% set co2_d = 90 if co2 > 1400 else 70 if co2 > 1000 else 50 if co2 > 800 else 35 if co2 > 600 else 20 %}
            {% set iaq_d = 90 if iaq > 350 else 70 if iaq > 250 else 50 if iaq > 150 else 35 if iaq > 100 else 20 %}
            {{ 'CO₂' if co2_d >= iaq_d else 'IAQ' }}
```

### Threshold Reference Table

| Level     | CO₂ (ppm) | IAQ Index | Fan Demand | Typical Scenario              |
| --------- | --------- | --------- | ---------- | ----------------------------- |
| Excellent | <600      | <100      | 20%        | Well-ventilated, unoccupied   |
| Good      | 600-800   | 100-150   | 35%        | Normal occupancy              |
| Moderate  | 800-1000  | 150-250   | 50%        | Several people, some activity |
| Poor      | 1000-1400 | 250-350   | 70%        | Crowded room, cooking         |
| Bad       | >1400     | >350      | 90%        | Many people, poor ventilation |

### Integration with Humidity Cascade

| Humidity Stage     | Base Fan Speed | Calculation                |
| ------------------ | -------------- | -------------------------- |
| Normal (Stage 0)   | AQ demand      | Direct from this sensor    |
| Elevated (Stage 1) | AQ + 20%       | `min(aq_demand + 20, 100)` |
| High (Stage 2)     | ≥70%           | `max(70, aq_demand)`       |
| Critical (Stage 3) | ≥90%           | `max(90, aq_demand)`       |

## Risks & Mitigations

| Risk                  | Likelihood | Impact | Mitigation                                  |
| --------------------- | ---------- | ------ | ------------------------------------------- |
| Sensor unavailable    | Low        | Medium | Fallback to 20% demand                      |
| IAQ calibration drift | Medium     | Low    | BME688 self-calibrates; 1000+ cycle burn-in |
| CO₂ inaccuracy        | Low        | Low    | SCD40 ±50ppm is good                        |
| Rapid oscillation     | Low        | Low    | Sensors already smoothed                    |

## Notes

- CO₂ 400ppm is outdoor baseline; anything below 600ppm is excellent
- IAQ from BME688 is a composite index (not ppm); higher = worse
- IAQ values stabilize after ~1000 measurement cycles (BME688 burn-in)
- This sensor provides the baseline; humidity cascade can override with higher values
