# Story 16.3: Humidity Rate Calculation

**Epic:** 16 - First Floor MEV Intelligent Control  
**Date:** January 16, 2026  
**Status:** Ready for Review  
**Story Points:** 3  
**Priority:** High

---

## User Story

As a **MEV control system**,  
I want **to calculate the rolling rate of humidity change (%/min) in ESPHome using a configurable window**,  
So that **fan speed and state machine logic can respond proactively to rising humidity, not just absolute levels**.

---

## Story Context

**Design Rationale (from Epic v5):**

The fan-speed-triggered state machine uses humidity rate as a key input:
- **Rising rate** (+%/min) increases fan speed demand (humidity is getting worse)
- **Falling rate** (-%/min) reduces fan speed demand (humidity is improving)
- **Zero rate** means steady state (position in window is sufficient)

This makes control **proactive** rather than purely reactive.

**Calculation Method:**

Use a rolling window (default 5 minutes, configurable 2-10 min via HA input_number) to compare current humidity against historical values:

```
rate (%/min) = (current_humidity - oldest_humidity) / time_elapsed_minutes
```

**ESPHome Implementation:**

- Calculation done in ESPHome using template sensor with lambda
- History stored in global variables (circular buffer)
- Window size read from HA input_number helper
- Sensor updates every minute
- Exposed to HA as `sensor.first_floor_mev_humidity_rate`
- Uses existing `first_floor_max_humidity` sensor defined in the room aggregation logic

**Integration:**

- Story 16.1 provides `first_floor_max_humidity` (aggregate sensor)
- This story adds rate calculation in ESPHome MEV component
- Story 16.4 will use this rate in the fan-speed-triggered state machine logic

### ESPHome Implementation

**Update to `components/mev.yaml`:**

```yaml
# Globals for humidity history (circular buffer)
globals:
  - id: ${mev_slug}_humidity_history_times
    type: std::vector<float>
    restore_value: no
    initial_value: 'std::vector<float>()'
  
  - id: ${mev_slug}_humidity_history_values
    type: std::vector<float>
    restore_value: no
    initial_value: 'std::vector<float>()'

# Sensor imports and calculations
sensor:
  # Import window configuration from HA
  - platform: homeassistant
    entity_id: input_number.mev_rate_window_minutes
    id: ${mev_slug}_rate_window
    internal: true

  # Humidity rate calculation (%/min)
  - platform: template
    name: "${mev_name} Humidity Rate"
    id: ${mev_slug}_humidity_rate
    unit_of_measurement: "%/min"
    accuracy_decimals: 2
    state_class: measurement
    icon: mdi:trending-up
    update_interval: 60s
    lambda: |-
      // Get current humidity and window size
      float current = id(${humidity_sensor}).state;
      float window_minutes = id(${mev_slug}_rate_window).state;
      
      // Validate inputs
      if (isnan(current) || isnan(window_minutes) || window_minutes <= 0) {
        return 0.0;
      }
      
      // Get current time (seconds since boot)
      float current_time = millis() / 1000.0;
      
      // Add current reading to history
      id(${mev_slug}_humidity_history_times).push_back(current_time);
      id(${mev_slug}_humidity_history_values).push_back(current);
      
      // Prune old readings (keep only those within window)
      float cutoff_time = current_time - (window_minutes * 60.0);
      size_t valid_start = 0;
      for (size_t i = 0; i < id(${mev_slug}_humidity_history_times).size(); i++) {
        if (id(${mev_slug}_humidity_history_times)[i] >= cutoff_time) {
          valid_start = i;
          break;
        }
      }
      
      // Remove old entries
      if (valid_start > 0) {
        id(${mev_slug}_humidity_history_times).erase(
          id(${mev_slug}_humidity_history_times).begin(),
          id(${mev_slug}_humidity_history_times).begin() + valid_start
        );
        id(${mev_slug}_humidity_history_values).erase(
          id(${mev_slug}_humidity_history_values).begin(),
          id(${mev_slug}_humidity_history_values).begin() + valid_start
        );
      }
      
      // Need at least 2 readings to calculate rate
      if (id(${mev_slug}_humidity_history_times).size() < 2) {
        return 0.0;
      }
      
      // Calculate rate: (current - oldest) / time_elapsed_minutes
      float oldest_value = id(${mev_slug}_humidity_history_values).front();
      float oldest_time = id(${mev_slug}_humidity_history_times).front();
      float time_elapsed_minutes = (current_time - oldest_time) / 60.0;
      
      if (time_elapsed_minutes <= 0) {
        return 0.0;
      }
      
      return (current - oldest_value) / time_elapsed_minutes;
```

**HA Input Number Helper (add to configuration.yaml or packages/):**

```yaml
input_number:
  mev_rate_window_minutes:
    name: "MEV Rate Calculation Window"
    min: 2
    max: 10
    step: 1
    unit_of_measurement: "min"
    icon: mdi:timer-sand
    initial: 5
```

## Risks & Mitigations

| Risk                      | Likelihood | Impact | Mitigation                                   |
| ------------------------- | ---------- | ------ | -------------------------------------------- |
| Sensor noise/spikes       | Medium     | Low    | Rolling window smooths transients            |
| History buffer growth     | Low        | Low    | Auto-pruned to window size (max 10 entries)  |
| Device restarts lose data | Medium     | Low    | History rebuilds over window period (~5 min) |
| Clock drift with millis() | Low        | Low    | Short windows minimize drift impact          |

## Notes

- Rate smoothing via rolling window is critical—instant rates would be too noisy
- 5-minute window balances responsiveness vs stability
- History stored in ESPHome globals (circular buffer, max 10 entries)
- Window size configurable via HA input_number for runtime adjustment
- Device restarts reset history—rebuilds over window period
- Story 16.4 will use rate > 0 as a key escalation signal (fan working hard but humidity still rising)
- millis() used for time tracking (resets on device restart but sufficient for rolling window)
