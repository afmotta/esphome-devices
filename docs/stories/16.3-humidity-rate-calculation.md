# Story 16.3: Humidity Rate Calculation

**Epic:** 16 - First Floor MEV Intelligent Control  
**Date:** January 16, 2026  
**Status:** Ready  
**Story Points:** 2  
**Priority:** High

---

## User Story

As a **MEV control system**,  
I want **to calculate the rolling rate of humidity change (%/min) using a configurable window**,  
So that **fan speed and state machine logic can respond proactively to rising humidity, not just absolute levels**.

---

## Story Context

**Design Rationale (from Epic v5):**

The fan-speed-triggered state machine uses humidity rate as a key input:
- **Rising rate** (+%/min) increases fan speed demand (humidity is getting worse)
- **Falling rate** (-%/min) reduces fan speed demand (humidity is improving)
- **Zero rate** means steady state (position in window is sufficient)

This makes control **proactive** rather than purely reactive.

**Calculation Method:**

Use a rolling window (default 5 minutes, configurable 2-10 min) to compare current humidity against historical values:

```
rate (%/min) = (current_humidity - oldest_humidity) / time_elapsed_minutes
```

**Integration:**

- Story 16.2 provides air quality demand (CO₂/IAQ → fan speed)
- This story adds humidity rate calculation as a template sensor in Home Assistant
- Story 16.4 will use this rate in the fan-speed-triggered state machine logic

## Acceptance Criteria

### Functional Requirements

1. **Rate Sensor Creation:**
   - [ ] Create `sensor.first_floor_humidity_rate` in HA template sensors
   - [ ] Unit: %/min
   - [ ] Update: Every 1 minute
   - [ ] State class: measurement

2. **Rolling Window Configuration:**
   - [ ] Create `input_number.mev_rate_window_minutes`
   - [ ] Range: 2-10 minutes
   - [ ] Default: 5 minutes
   - [ ] Step: 1 minute

3. **Calculation Logic:**
   - [ ] Store humidity history with timestamps in sensor attributes
   - [ ] Keep only readings within the configured window
   - [ ] Calculate linear rate: (current - oldest) / time_elapsed
   - [ ] Return 0 if insufficient data (<2 readings)

4. **Rate Characteristics:**
   - [ ] Positive rate = humidity rising
   - [ ] Negative rate = humidity falling
   - [ ] Zero rate = steady state
   - [ ] Smoothed over window (not instant changes)

### Technical Requirements

5. **HA Template Sensor:**
   - [ ] Trigger: time_pattern every 1 minute
   - [ ] State calculation with proper null/error handling
   - [ ] Attributes store history array

6. **History Management:**
   - [ ] Add current reading with timestamp each minute
   - [ ] Prune readings older than window
   - [ ] Preserve history across sensor updates

7. **Error Handling:**
   - [ ] Graceful fallback if `first_floor_max_humidity` unavailable
   - [ ] Return 0 if not enough historical data
   - [ ] Handle division by zero

## Technical Implementation

### Files to Create (in HA configuration.yaml or packages/)

**1. Input Number Helper:**

```yaml
input_number:
  mev_rate_window_minutes:
    name: "MEV Rate Calculation Window"
    min: 2
    max: 10
    step: 1
    unit_of_measurement: "min"
    icon: mdi:timer-sand
    initial: 5
```

**2. Template Sensor:**

```yaml
template:
  - trigger:
      - platform: time_pattern
        minutes: "/1"
    sensor:
      - name: "First Floor Humidity Rate"
        unique_id: first_floor_humidity_rate
        unit_of_measurement: "%/min"
        state_class: measurement
        icon: mdi:trending-up
        state: >
          {% set window = states('input_number.mev_rate_window_minutes') | int(5) %}
          {% set current = states('sensor.first_floor_max_humidity') | float(none) %}
          {% set history = state_attr('sensor.first_floor_humidity_rate', 'history') | default([]) %}
          
          {# Add current reading to history #}
          {% set ns = namespace(hist = history + [{'time': now().timestamp(), 'value': current}]) %}
          
          {# Keep only readings within window #}
          {% set cutoff = now().timestamp() - (window * 60) %}
          {% set ns.hist = ns.hist | selectattr('time', '>=', cutoff) | list %}
          
          {# Calculate rate if we have enough data #}
          {% if ns.hist | length >= 2 and current is not none %}
            {% set oldest = ns.hist | first %}
            {% set dt_min = (now().timestamp() - oldest.time) / 60 %}
            {% if dt_min > 0 %}
              {{ ((current - oldest.value) / dt_min) | round(2) }}
            {% else %}
              0
            {% endif %}
          {% else %}
            0
          {% endif %}
        attributes:
          history: >
            {% set window = states('input_number.mev_rate_window_minutes') | int(5) %}
            {% set current = states('sensor.first_floor_max_humidity') | float(none) %}
            {% set history = state_attr('sensor.first_floor_humidity_rate', 'history') | default([]) %}
            {% set cutoff = now().timestamp() - (window * 60) %}
            {{ (history + [{'time': now().timestamp(), 'value': current}]) 
               | selectattr('time', '>=', cutoff) | list }}
```

## Notes

- Rate smoothing via rolling window is critical—instant rates would be too noisy
- 5-minute window balances responsiveness vs stability
- The `history` attribute is for debugging/diagnostics—the state is what automation uses
- Story 16.4 will use rate > 0 as a key escalation signal (fan working hard but humidity still rising)
