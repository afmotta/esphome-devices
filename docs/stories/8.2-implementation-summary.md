## Interface Contract Compliance

### Required Globals Exposed

```yaml
# State enum: 0=Normal, 1=Active, 2=Recovering
global:
  - id: ${zone_slug}_window_state
    type: int
    initial_value: "0"

# Priority: 2 (second highest, below emergency=1)
global:
  - id: ${zone_slug}_window_priority
    type: int
    initial_value: "2"
```

### State Machine Transitions

```
Normal (0) ──[window open + mode match + 180s timeout]──> Active (1)

Active (1) ──[window closes]──> Recovering (2)
       └────[mode changes to non-shutdown]──> Recovering (2)

Recovering (2) ──[60s stability]──> Normal (0)
           └────[window re-opens + mode match]──> Active (1)
```

**Unique Feature:** Two paths from Active→Recovering (window close OR mode change)

---

## Mode-Awareness Logic

**Design Decision:** Window condition retains mode-awareness (not "dumb sensor").

**Rationale:** Window open during heating is wasteful but not critical; window open during cooling is highly wasteful. Condition understands its own trigger semantics without coordinator needing mode-specific rules.

### Supported Mode Configurations

| Configuration        | Behavior                                       |
| -------------------- | ---------------------------------------------- |
| `"cooling, heating"` | Triggers in both HEAT and COOL modes (typical) |
| `"cooling"`          | Triggers only in COOL mode                     |
| `"heating"`          | Triggers only in HEAT mode                     |
| `"auto"`             | Triggers only in AUTO mode                     |
| `"heat_cool"`        | Triggers only in HEAT_COOL mode                |
| `""` (empty)         | Never triggers (not recommended)               |

**Mode Parsing:** Simple string matching via `std::string::find()` on comma-separated list.

## Coordinator Integration Pattern

The coordinator (Story 8.3) will read these globals:

```cpp
// Read window condition state
int window_state = id(soggiorno_window_state);
int window_priority = id(soggiorno_window_priority);

// Check if Active or Recovering
if (window_state == 1 || window_state == 2) {
  // Force PID OFF (shutdown active)
  if (pid->mode != climate::CLIMATE_MODE_OFF) {
    auto call = pid->make_call();
    call.set_mode(climate::CLIMATE_MODE_OFF);
    call.perform();
  }
}
```

**Critical:** Coordinator handles priority hierarchy when multiple conditions are Active.

## Known Limitations

1. **Mode parsing:** Simple string matching (no whitespace tolerance)
2. **Sensor failure:** Treats unavailable sensor as "closed" (fail-safe)
3. **Recovery from mode change:** Coordinator still keeps PID OFF during recovery
4. **Grace period precision:** Text sensor shows ~5s granularity
