### Testing Checklist Content

```markdown
# Epic 6: MEV Testing Checklist

## Pre-Installation (Bench Testing)

### Basic Connectivity
- [ ] Board powers on (LED indicators active)
- [ ] WiFi/Ethernet connection established
- [ ] Board appears in HA ESPHome integration
- [ ] All entities visible in HA

### Relay Testing (With Multimeter)
- [ ] Power relay: Continuity check NO/NC contacts
- [ ] Winter/Summer relay: Continuity check
- [ ] Dehumidifier relay: Continuity check
- [ ] Cooling relay: Continuity check
- [ ] All relays respond to HA switch commands within 1 second

### 0-10V Output Testing (With Multimeter)
- [ ] 0% fan speed = 0.0V ±0.2V
- [ ] 25% fan speed = 2.5V ±0.2V
- [ ] 50% fan speed = 5.0V ±0.2V
- [ ] 75% fan speed = 7.5V ±0.2V
- [ ] 100% fan speed = 10.0V ±0.2V
- [ ] Response time < 1 second
- [ ] Voltage stable (no fluctuation > 0.1V)

### Alarm Input Testing
- [ ] Open circuit state = binary_sensor OFF
- [ ] Closed circuit state = binary_sensor ON
- [ ] State change reflects in HA within 1 second
- [ ] No spurious triggering (5 minute observation)

## Post-Installation (With MEV Unit)

### MEV Response Verification
- [ ] Power relay toggles MEV unit on/off
- [ ] Winter/Summer mode changes MEV behavior as expected
- [ ] Dehumidifier activates when relay closed
- [ ] Cooling integration activates when relay closed
- [ ] Fan speed varies smoothly with 0-10V input
- [ ] Alarm input triggers when MEV reports fault

### HA Integration Validation
- [ ] All entities show correct states in HA
- [ ] Entity names and friendly names correct
- [ ] Device appears in HA device registry as "MEV Primo Piano"
- [ ] Icon and device class assignments appropriate

### Autonomous Operation Test
- [ ] Set MEV to known state (e.g., 50% fan speed, power on)
- [ ] Disconnect HA (stop HA or disconnect network)
- [ ] Wait 5+ minutes
- [ ] Verify MEV continues operating at last commanded state
- [ ] Reconnect HA
- [ ] Verify HA regains control within 1 minute
```
