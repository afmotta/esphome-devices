## User Story

**As a** homeowner  
**I want** a KC868-A6 board configured for MEV control  
**So that** I can expose MEV hardware controls to Home Assistant

## Description

Create the foundational device configuration for the MEV board, including base board setup, device-level assembly, and local/remote configuration files. This establishes the hardware abstraction layer following the Epic 5 architecture pattern (HA-driven control, exposed entities only).

## Technical Implementation Notes

### File Structure
```
devices/
  mev-primo-piano.yaml      # Main device assembly
locals/
  mev-primo-piano.yaml      # Secrets and local substitutions
remotes/
  mev-primo-piano.yaml      # Remote configuration support
```

### Device Configuration Pattern
```yaml
# devices/mev-primo-piano.yaml
substitutions:
  device_name: mev-primo-piano
  friendly_name: MEV Primo Piano
  device_description: "First Floor Mechanical Extract Ventilation Control"

packages:
  base: !include ../boards/a6.yaml
  wifi: !include ../boards/wifi.yaml
  mev: !include ../components/mev.yaml
    vars:
      mev_slug: "mev_primo_piano"
      mev_name: "MEV Primo Piano"
```
