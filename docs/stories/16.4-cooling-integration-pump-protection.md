# Story 16.4: Cooling Integration with Pump Protection

**Epic:** 16 - First Floor MEV Intelligent Control  
**Date:** January 15, 2026  
**Status:** Ready  
**Story Points:** 3  
**Priority:** High

---

## User Story

As a **MEV control system**,  
I want **cooling pump activation only in Critical stage with minimum ON/OFF times**,  
So that **the pump is protected from short-cycling while providing maximum dehumidification when needed**.

---

## Story Context

**From Brainstorming:**

- MEV cooling relay directly controls a circulation pump
- Pump draws chilled water from Heat Pump buffer
- Cold water only available in Summer Mode
- Pump short-cycling damages the pump and wastes energy
- Agreed protection: 5 min ON minimum, 10 min OFF minimum

**Hardware (Epic 6):**

- `switch.first_floor_mev_cooling` — Controls the circulation pump relay
- `binary_sensor.summer_mode` — Indicates when HP buffer has cold water

---

## Acceptance Criteria

### Functional Requirements

1. **Cooling Prerequisites:**
   - [ ] Only activates when Stage 3 (Critical) is active
   - [ ] Only activates when Summer Mode is ON
   - [ ] Never activates in Winter Mode (no cold water)

2. **Pump Protection Timers:**
   - [ ] Create `timer.mev_cooling_pump_min_on` (5 minutes)
   - [ ] Create `timer.mev_cooling_pump_min_off` (10 minutes)

3. **Start Logic:**
   - [ ] Pump can START only if:
     - Stage = Critical AND
     - Summer Mode = ON AND
     - min_off timer is idle (finished or never started)

4. **Stop Logic:**
   - [ ] Pump can STOP only if:
     - Stage < Critical (exited Critical) AND
     - min_on timer is idle (finished or never started)

5. **Timer Behavior:**
   - [ ] When pump turns ON → start min_on timer
   - [ ] When pump turns OFF → start min_off timer
   - [ ] Timers prevent opposite action until they finish

### Technical Requirements

6. **HA Timer Helpers:**
   - [ ] Create in HA (Settings → Helpers → Timer)
   - [ ] IDs match the names above

7. **Integration with Main Automation:**
   - [ ] Story 16.6 automation must check timer states
   - [ ] Pump control respects timer protection

### Pump Control Logic (for Story 16.6)

**Pump START logic:**
```yaml
# Within the main automation action for Stage 3 (Critical)
- if:
    # All conditions must be true to start pump
    - condition: state
      entity_id: binary_sensor.summer_mode
      state: "on"
    - condition: state
      entity_id: timer.mev_cooling_pump_min_off
      state: "idle"
  then:
    - service: switch.turn_on
      target:
        entity_id: switch.first_floor_mev_cooling
    - service: timer.start
      target:
        entity_id: timer.mev_cooling_pump_min_on
```

**Pump STOP logic:**
```yaml
# Within the main automation action for Stage < 3
- if:
    # Can only stop if min_on time has passed
    - condition: state
      entity_id: timer.mev_cooling_pump_min_on
      state: "idle"
  then:
    - service: switch.turn_off
      target:
        entity_id: switch.first_floor_mev_cooling
    - service: timer.start
      target:
        entity_id: timer.mev_cooling_pump_min_off
```

### Timing Scenarios

| Scenario | Behavior |
|----------|----------|
| Enter Critical, Summer ON | Pump starts, min_on timer starts (5min) |
| Exit Critical after 2 min | Pump stays ON until min_on completes (3 more min) |
| Exit Critical after 7 min | Pump stops immediately, min_off starts (10min) |
| Re-enter Critical after 5 min | min_off still running; pump cannot start |
| Re-enter Critical after 12 min | min_off complete; pump can start |

### Safety Scenarios

| Scenario | Expected Behavior |
|----------|-------------------|
| Winter Mode, Stage Critical | Pump does NOT start (no cold water) |
| Summer→Winter while pump ON | Pump stays ON until min_on complete, then stops |
| HA restart while pump ON | Timers reset; pump stays in current state |
| Manual override via switch | Works; timers don't block manual control |

## Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|------|------------|--------|------------|
| Pump runs in winter | Low | High | Summer Mode prerequisite check |
| Pump short-cycles | Low | High | Timer-based protection |
| Timer lost on HA restart | Medium | Low | Pump stays in current state |
| Stuck in Critical | Low | Medium | De-escalation automations (Story 16.3) |

## Notes

- Pump protection is critical for hardware longevity
- Timers are HA-side; ESPHome switch has no built-in protection
- Manual override (via HA UI or switch entity) bypasses timers—by design
- If HA restarts, timers reset but pump state persists in ESPHome
- Consider adding pump runtime sensor for diagnostics (future enhancement)
