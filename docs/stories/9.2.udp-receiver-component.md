## Story

**As a** system architect,  
**I want** a reusable UDP receiver component that consumes UDP broadcasts on slave boards,  
**so that** slave boards can receive supply temperatures and climate mode data directly from the master board without Home Assistant intermediation.

---

## Acceptance Criteria

1. Component package `components/udp_receiver.yaml` is created following ESPHome package composition pattern
2. Component accepts configurable variables: `listen_port` (UDP port), `master_ip` (optional filter), `sensor_mappings` (dict mapping remote sensor IDs to local sensor IDs)
3. Component listens for UDP packets on specified port (default 6053) and parses JSON payloads
4. Component creates internal sensors for each mapped remote sensor (e.g., `supply_temp_piano_terra_udp`)
5. Component updates internal sensors with values from parsed JSON packets
6. Component handles malformed JSON gracefully (logs error, doesn't crash)
7. Component includes feature flag support via `use_udp` substitution variable (default: false)
8. When `use_udp: false`, component creates stub sensors that always report NaN
9. Component includes diagnostic sensors: `udp_packets_received` (counter), `udp_last_received` (timestamp), `udp_parse_errors` (counter)
10. Component successfully compiles on KC868-A16 boards (distribuzione boards) without firmware size overflow

### UDP Listening Implementation

**ESP-IDF UDP Socket Approach:**
```yaml
# In on_boot or setup priority
lambda: |-
  #include <sys/socket.h>
  #include <netinet/in.h>
  
  int sock = socket(AF_INET, SOCK_DGRAM, IPPROTO_UDP);
  struct sockaddr_in listen_addr;
  listen_addr.sin_family = AF_INET;
  listen_addr.sin_port = htons(${listen_port});
  listen_addr.sin_addr.s_addr = INADDR_ANY;
  
  bind(sock, (struct sockaddr*)&listen_addr, sizeof(listen_addr));
  
  // Non-blocking receive in interval or component loop
  char buffer[1024];
  struct sockaddr_in src_addr;
  socklen_t src_len = sizeof(src_addr);
  int len = recvfrom(sock, buffer, sizeof(buffer), MSG_DONTWAIT,
                     (struct sockaddr*)&src_addr, &src_len);
  
  if (len > 0) {
    buffer[len] = '\0';
    parse_json_and_update_sensors(buffer);
  }
```

### JSON Parsing Strategy

**Manual Parsing (Minimal firmware size):**
```cpp
// Simple key-value extraction
std::string find_sensor_value(const char* json, const char* sensor_id) {
  // Find "id":"sensor_id" and extract adjacent "value":123.45
  // Brittle but compact
}
```

**ArduinoJson Library (More robust):**
```cpp
#include <ArduinoJson.h>

DynamicJsonDocument doc(1024);
DeserializationError error = deserializeJson(doc, json_string);
if (!error) {
  JsonArray sensors = doc["sensors"];
  for (JsonObject sensor : sensors) {
    const char* id = sensor["id"];
    float value = sensor["value"];
    // Update corresponding sensor
  }
}
```

**Recommendation:** Start with ArduinoJson if firmware size allows (more maintainable).

### Sensor Mapping Pattern

Component variables:
```yaml
vars:
  sensor_mappings:
    - remote_id: "supply_temp_piano_terra"
      local_id: "supply_temp_piano_terra_udp"
      unit: "°C"
    - remote_id: "climate_mode"
      local_id: "climate_mode_udp"
```

Generated sensors:
```yaml
sensor:
  - platform: template
    id: supply_temp_piano_terra_udp
    name: "Supply Temp Piano Terra (UDP)"
    internal: true
    unit_of_measurement: "°C"
    accuracy_decimals: 1
    lambda: |-
      return id(udp_received_values)["supply_temp_piano_terra"];
```
