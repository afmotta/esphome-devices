# Story 16.3: Humidity Cascade State Machine

**Epic:** 16 - First Floor MEV Intelligent Control  
**Date:** January 15, 2026  
**Status:** Ready  
**Story Points:** 3  
**Priority:** High

---

## User Story

As a **MEV control system**,  
I want **a 4-stage humidity cascade that escalates equipment activation based on humidity severity**,  
So that **minor humidity spikes use minimal intervention while severe humidity gets aggressive response**.

---

## Story Context

**Design Decision (from Brainstorming):**

- 4 stages: Normal → Elevated → High → Critical
- Each stage adds more equipment and higher fan speeds
- Hysteresis prevents rapid oscillation between stages
- Time-delayed transitions ensure stable conditions before stage changes

**Equipment Escalation:**

| Stage        | Fan Speed | Dehumidifier | Cooling Pump   |
| ------------ | --------- | ------------ | -------------- |
| Normal (0)   | AQ demand | OFF          | OFF            |
| Elevated (1) | AQ + 20%  | OFF          | OFF            |
| High (2)     | ≥70%      | ON           | OFF            |
| Critical (3) | ≥90%      | ON           | ON (if summer) |

---

## Acceptance Criteria

### Functional Requirements

1. **State Machine Entity:**
   - [ ] Create `input_select.mev_humidity_stage`
   - [ ] Options: normal, elevated, high, critical
   - [ ] Initial state: normal

2. **Escalation Thresholds:**
   - [ ] Normal → Elevated: humidity ≥55% for 5 minutes
   - [ ] Elevated → High: humidity ≥65% for 10 minutes
   - [ ] High → Critical: humidity ≥75% for 10 minutes

3. **De-escalation Thresholds (with hysteresis):**
   - [ ] Elevated → Normal: humidity <55% for 10 minutes
   - [ ] High → Normal: humidity <60% for 15 minutes
   - [ ] Critical → Normal: humidity <65% for 20 minutes

4. **Stage Actions:**
   - [ ] Normal: Fan at AQ demand only
   - [ ] Elevated: Fan at AQ demand +20% (capped at 100%)
   - [ ] High: Fan ≥70%, Dehumidifier ON
   - [ ] Critical: Fan ≥90%, Dehumidifier ON, Cooling ON (summer only)

### Technical Requirements

5. **Input Select Helper:**
   - [ ] Create in HA (Settings → Helpers → Input Select)
   - [ ] ID: `input_select.mev_humidity_stage`

6. **Transition Automations (6 total):**
   - [ ] Normal → Elevated automation
   - [ ] Elevated → High automation
   - [ ] High → Critical automation
   - [ ] Elevated → Normal automation
   - [ ] High → Normal automation
   - [ ] Critical → Normal automation

### Transition Automations

**1. Normal → Elevated:**
```yaml
automation:
  - alias: "MEV Stage: Normal → Elevated"
    id: mev_stage_normal_to_elevated
    trigger:
      - platform: numeric_state
        entity_id: sensor.first_floor_max_humidity
        above: 55
        for:
          minutes: 5
    condition:
      - condition: state
        entity_id: input_select.mev_humidity_stage
        state: "normal"
    action:
      - service: input_select.select_option
        target:
          entity_id: input_select.mev_humidity_stage
        data:
          option: "elevated"
      - service: notify.persistent_notification
        data:
          title: "MEV Humidity Stage Changed"
          message: "Stage: Normal → Elevated (humidity >55%)"
```

### Threshold Summary

| Transition        | Entry Threshold | Entry Delay | Exit Threshold | Exit Delay |
| ----------------- | --------------- | ----------- | -------------- | ---------- |

| Elevated → High   | ≥65%            | 10 min      | <60%           | 15 min     |
| High → Critical   | ≥75%            | 10 min      | <65%           | 20 min     |

## Risks & Mitigations

| Risk                  | Likelihood | Impact | Mitigation                            |
| --------------------- | ---------- | ------ | ------------------------------------- |
| Stage oscillation     | Low        | Medium | Hysteresis in exit thresholds         |
| Shower spike→Critical | Medium     | Low    | 10-minute delays prevent overreaction |
| Stuck in high stage   | Low        | Medium | De-escalation automations             |
| Manual override lost  | Low        | Low    | Can manually set input_select         |

## Notes

- De-escalation always goes to "normal" (not stepwise) for simplicity
- Exit thresholds are lower than entry thresholds (hysteresis)
- Exit delays increase with stage severity (more aggressive = more cautious exit)
- Notifications help with debugging during commissioning
