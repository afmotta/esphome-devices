# Story 3.1: Remove Board-to-Board Modbus from gruppo-miscelazione

## Status
Ready for Review

## Story
**As a** system architect,  
**I want** to remove board-to-board Modbus master functionality from gruppo-miscelazione while keeping sensor exposure,  
**so that** the mixing valve board operates independently and coordinates through Home Assistant instead of direct Modbus slave communication.

## Acceptance Criteria

1. Remove `modbus_master` package from `devices/gruppo-miscelazione.yaml`
2. Remove `modbus_master_registers` package from `devices/gruppo-miscelazione.yaml`
3. Keep Dallas temperature sensors exposed to Home Assistant via ESPHome API
4. Remove master-specific automation and scripts for slave coordination
5. Verify mixing valve PID control still functions correctly
6. Verify Dallas sensors are visible in Home Assistant
7. Configuration validates and compiles without errors
8. No regression in mixing valve temperature control behavior

## Dev Notes

### Relevant Source Tree
- **Device file**: `devices/gruppo-miscelazione.yaml`
- **Master components** (to remove): 
  - `components/modbus_master.yaml`
  - `components/modbus_master_registers.yaml`
- **Sensors to preserve**: Dallas temperature sensors (1-wire bus)
- **PID controllers**: Already simplified in Epic 2, no changes needed here

### Architecture Context
**Current Role**: gruppo-miscelazione is the Modbus master board (A6) that:
- Exposes Dallas temperature sensors via Modbus registers to slave boards
- Provides climate mode coordination via Modbus registers
- Controls two mixing valve circuits with PID controllers

**After Epic 3.1**: gruppo-miscelazione becomes independent board that:
- Still exposes Dallas sensors, but via ESPHome API to Home Assistant only
- Removes all Modbus master/slave coordination logic
- Mixing valves operate independently, coordinated by HA

### Important Notes from Previous Stories
- Epic 2 already simplified PID architecture (single entity per circuit)
- Dallas sensors use 1-wire protocol (`one_wire_01` bus), not Modbus
- PID controllers reference Dallas sensors by ID (e.g., `dallas_0x81000000b3e6f628`)
- Board uses A6 platform with DAC outputs for mixing valve control

### Key Preservation Points
1. **Dallas Sensors**: Keep all Dallas sensor configurations unchanged
2. **PID Controllers**: No changes to PID climate entities
3. **Relay/DAC Outputs**: Keep all output configurations
4. **ESPHome API**: Ensure enabled for HA integration
