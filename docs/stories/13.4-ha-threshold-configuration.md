# Story 13.4: HA Threshold Configuration

**Epic:** 13 - Ground Floor Hybrid Radiant-Fancoil Cooling  
**Date:** December 26, 2025  
**Status:** Ready  
**Story Points:** 1  
**Priority:** Medium

---

## User Story

As a **system owner**,  
I want **to adjust the fancoil boost threshold via Home Assistant at runtime**,  
So that **I can tune the system without recompiling firmware**.

---

## Story Context

**Decision from Elicitation:**
- Single threshold for entire ground floor
- HA `input_number` entity for runtime adjustment
- ESPHome fallback to 4.0°C if HA unavailable
- Hysteresis = threshold / 2

**Use Case:**
- Summer starts → threshold at 4°C (default)
- Peak heat wave → lower to 3°C for more aggressive fancoil activation
- Mild days → raise to 5°C to maximize radiant comfort

## Technical Implementation

### Home Assistant Configuration

**Option A: UI Helper (Recommended)**
1. Settings → Devices & Services → Helpers
2. Create Number helper with parameters above

**Option B: configuration.yaml**
```yaml
input_number:
  ground_floor_fancoil_boost_threshold:
    name: "Ground Floor Fancoil Boost Threshold"
    min: 1.0
    max: 10.0
    step: 0.5
    initial: 4.0
    unit_of_measurement: "°C"
    icon: mdi:thermometer-alert
```

### ESPHome Configuration

**In ground-floor.yaml or climate-control.yaml:**
```yaml
number:
  - platform: homeassistant
    id: fancoil_boost_threshold
    entity_id: input_number.ground_floor_fancoil_boost_threshold
    name: "Fancoil Boost Threshold"
    internal: true  # Don't re-expose to HA

globals:
  - id: fancoil_boost_threshold_fallback
    type: float
    initial_value: '4.0'

sensor:
  - platform: template
    id: effective_fancoil_boost_threshold
    name: "Effective Fancoil Boost Threshold"
    unit_of_measurement: "°C"
    accuracy_decimals: 1
    lambda: |-
      if (id(fancoil_boost_threshold).has_state()) {
        return id(fancoil_boost_threshold).state;
      }
      ESP_LOGW("threshold", "HA unavailable, using fallback: %.1f°C", 
               id(fancoil_boost_threshold_fallback));
      return id(fancoil_boost_threshold_fallback);
    update_interval: 30s
```

### Usage in Coordinator (Story 13.5)

```yaml
# Coordinator lambda will use:
float threshold = id(effective_fancoil_boost_threshold).state;
float hysteresis = threshold / 2.0;

// Boost activation
if (delta > threshold) { ... }

// Boost deactivation  
if (delta < hysteresis) { ... }
```
