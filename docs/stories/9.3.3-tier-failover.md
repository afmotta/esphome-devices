## Story

**As a** system operator,  
**I want** an enhanced 3-tier sensor failover system (UDP → HA → Emergency),  
**so that** slave boards automatically fall back to Home Assistant sensors when UDP is unavailable, and then to emergency shutdown when both tiers fail.

---

## Acceptance Criteria

1. Component package `components/sensor_failover_3tier.yaml` is created following ESPHome package composition pattern
2. Component implements 3-tier failover logic: Tier 1 (UDP sensor) → Tier 2 (HA sensor) → Tier 3 (Emergency/NaN)
3. Component accepts configurable variables: `udp_sensor_id`, `ha_sensor_id`, `abstracted_sensor_id`, `emergency_timeout` (seconds)
4. Component creates abstracted sensor that PIDs consume (e.g., `supply_temp_piano_terra_abstracted`)
5. Tier 1 (UDP) is active when UDP sensor has valid value (not NaN)
6. Tier 2 (HA) is active when UDP sensor is NaN but HA sensor has valid value
7. Tier 3 (Emergency) is active when both UDP and HA sensors are NaN for longer than `emergency_timeout`
8. Component includes diagnostic `text_sensor` showing active tier: "UDP", "HA", or "Emergency"
9. Component integrates with existing Epic 8 emergency condition components
10. Failover transitions are logged to ESPHome log with tier change reasons

### Failover Logic Implementation

**Abstracted Sensor Lambda:**
```yaml
sensor:
  - platform: template
    id: ${abstracted_sensor_id}
    name: "${zone_name} Temperature (Abstracted)"
    internal: true
    unit_of_measurement: "°C"
    accuracy_decimals: 1
    update_interval: 10s
    lambda: |-
      // Tier 1: UDP sensor
      if (!isnan(id(${udp_sensor_id}).state)) {
        id(${zone}_sensor_tier).publish_state("UDP");
        ESP_LOGD("failover", "Using Tier 1 (UDP): %.1f", id(${udp_sensor_id}).state);
        return id(${udp_sensor_id}).state;
      }
      
      // Tier 2: HA sensor
      if (!isnan(id(${ha_sensor_id}).state)) {
        id(${zone}_sensor_tier).publish_state("HA");
        ESP_LOGD("failover", "Using Tier 2 (HA): %.1f", id(${ha_sensor_id}).state);
        return id(${ha_sensor_id}).state;
      }
      
      // Tier 3: Emergency (both unavailable)
      id(${zone}_sensor_tier).publish_state("Emergency");
      ESP_LOGW("failover", "Tier 3 (Emergency): No valid sensors");
      return NAN;
```

**Diagnostic Text Sensor:**
```yaml
text_sensor:
  - platform: template
    id: ${zone}_sensor_tier
    name: "${zone_name} Active Sensor Tier"
```

### Integration with Epic 8 Architecture

**Epic 8 Emergency Condition Component:**
- Monitors `${zone}_room_temp_abstracted` sensor
- Triggers emergency when sensor is NaN for > 180s
- Sets global `${zone}_emergency_state` to Active (1)

**With Epic 9 3-Tier Failover:**
- Abstracted sensor uses UDP/HA/Emergency tiers
- Emergency condition monitors the abstracted sensor (same interface)
- When Tier 3 (Emergency) active → abstracted sensor is NaN → emergency condition triggers
- Coordinator sees emergency condition active → forces PID OFF

**No changes required to Epic 8 components** - they continue to work with abstracted sensor.

### Tier Priority Rules

| UDP Sensor | HA Sensor   | Active Tier        | Abstracted Value | Emergency Condition |
| ---------- | ----------- | ------------------ | ---------------- | ------------------- |
| Valid      | Valid       | Tier 1 (UDP)       | UDP value        | Normal              |
| Valid      | NaN         | Tier 1 (UDP)       | UDP value        | Normal              |
| NaN        | Valid       | Tier 2 (HA)        | HA value         | Normal              |
| NaN        | NaN (<180s) | Tier 2 (HA)        | NaN (recovering) | Normal              |
| NaN        | NaN (>180s) | Tier 3 (Emergency) | NaN              | Active              |

### Testing Standards

1. **Tier 1 Test:** Set UDP sensor = 25.0, HA sensor = 26.0 → Expect abstracted = 25.0, tier = "UDP"

3. **Tier 3 Test:** Set UDP sensor = NaN, HA sensor = NaN, wait 181s → Expect abstracted = NaN, tier = "Emergency"

**Integration Testing:**
