# Story 8.5 Tier 2 Migration Plan - Ground Floor Rooms

**Date:** November 1, 2025  
**Epic:** 8 - Unified State Machine Architecture  
**Phase:** Tier 2 Rollout (Ground Floor Mixed Rooms)  
**Rooms:** 4 rooms (2 fancoil rooms, 2 radiant-only rooms)

---

## Overview

This document details the Tier 2 migration plan for ground floor rooms from Epic 5/7 to Epic 8 architecture. Ground floor includes **mixed equipment types**, requiring different Epic 8 component combinations per room.

---

## Tier 2 Rooms - Architecture Matrix

| Room               | Equipment         | Epic 8 Components                | Priority | Notes                      |
| ------------------ | ----------------- | -------------------------------- | -------- | -------------------------- |
| **Cucina**         | Radiant + Fancoil | Emergency + Window + Coordinator | Medium   | Full climate control       |
| **Sala Pranzo**    | Fancoil only      | Window + Coordinator             | Medium   | Uses Soggiorno temp sensor |
| **Locale Tecnico** | Fancoil only      | Emergency + Window + Coordinator | Low      | Technical room             |
| **Bagno Terra**    | Radiant only      | Emergency + Coordinator          | Low      | Bathroom, lower occupancy  |
| **Anticamera**     | Radiant only      | Emergency + Coordinator          | Low      | Hallway, minimal needs     |

**Key Insight:** Sala Pranzo is a **unique case** - fancoil-only room with no temperature sensor of its own (uses Soggiorno's sensor). It needs window condition but NOT emergency condition.

### Step 8: Functional Testing (Per Room Type)

**For Cucina (Emergency + Window):**
1. Emergency test: Disconnect HA temperature sensor (3 min timeout)
2. Window test: Open window with fancoil in cooling/heating (3 min grace)
3. Priority test: Both conditions active (emergency priority 1 wins)

**For Sala Pranzo (Window only):**
1. Window test: Open window with fancoil active (3 min grace)
2. Mode-awareness: Open window with fancoil OFF (should ignore)
3. Coordinator observability: Status sensor updates

**For Locale Tecnico (Emergency + Window):**
1. Emergency test: Disconnect HA temperature sensor (3 min timeout)
2. Window test: Open window with fancoil in cooling/heating (3 min grace)
3. Priority test: Both conditions active (emergency priority 1 wins)

**For Bagno Terra & Anticamera (Emergency only):**
1. Emergency test: Disconnect HA temperature sensor (3 min timeout)
2. Recovery test: Restore sensor (1 min recovery)
3. Coordinator observability: Status sensor updates

### Step 9: Overnight Soak Test (12+ hours per room)
1. Monitor ESPHome logs for errors
2. Verify temperature control quality (±1°C)
3. Check coordinator status stability
4. Confirm no unexpected state transitions

### Step 10: Sign-Off
- [ ] Functional tests passed (condition-specific)
- [ ] Overnight soak test passed (12h+ stable)
- [ ] Climate quality maintained (±1°C)
- [ ] No errors in logs
- [ ] Coordinator status accurate
- ✅ Proceed to next room

## Risk Mitigation

**Risk:** Mixed configurations on same device (Soggiorno already Epic 8)

**Mitigation:**
- Soggiorno validated stable (Story 8.4 complete)
- Stagger migrations 1 room/day
- Each room independently validated
- Device-level monitoring (ESP32 free heap)

**Risk:** Sala Pranzo unique architecture (fancoil-only, shared sensor)

**Mitigation:**
- Test window condition thoroughly
- Verify coordinator controls correct PID
- Monitor Soggiorno sensor reliability (affects Sala Pranzo)

**Stop-Gate Criteria:**
- ❌ Any room requires rollback after soak test
- ❌ Climate quality degrades >2°C in any room
- ❌ Soggiorno destabilizes (shared device)
- ❌ Sala Pranzo issues with shared sensor
