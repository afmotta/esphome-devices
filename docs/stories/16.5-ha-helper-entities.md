# Story 16.5: HA Helper Entities

**Epic:** 16 - First Floor MEV Intelligent Control  
**Date:** January 15, 2026  
**Updated:** January 16, 2026  
**Status:** Ready  
**Story Points:** 2  
**Priority:** High

---

## User Story

As a **system integrator**,  
I want **all required Home Assistant helper entities created and configured**,  
So that **the MEV control automations have the state storage and runtime-configurable parameters they need**.

---

## Story Context

This story consolidates all HA helper entity creation into a single checklist. These helpers are referenced by Stories 16.2, 16.3, 16.4, and 16.6.

**Entity Types:**
- **Input Numbers:** Runtime-configurable parameters (humidity bounds, fan thresholds, delays)
- **Input Select:** State machine storage (fan_only / dehumidifying / cooling)
- **Template Sensors:** Diagnostic calculations (air quality demand, humidity demand, final fan speed)

---

## Acceptance Criteria

### Required Helper Entities

1. **Input Select (State Machine):**
   - [ ] `input_select.mev_humidity_state`
     - Options: fan_only, dehumidifying, cooling
     - Initial: fan_only

2. **Input Numbers (Humidity Window):**
   - [ ] `input_number.mev_humidity_upper_bound` (50-70%, default 60%)
   - [ ] `input_number.mev_humidity_lower_bound` (30-50%, default 40%)

3. **Input Numbers (Fan Thresholds):**
   - [ ] `input_number.mev_escalation_fan_threshold` (50-90%, default 70%)
   - [ ] `input_number.mev_escalation_fan_threshold_cooling` (60-95%, default 80%)
   - [ ] `input_number.mev_deescalation_fan_threshold` (20-50%, default 30%)
   - [ ] `input_number.mev_minimum_fan_speed` (10-40%, default 20%)

4. **Input Numbers (Timing):**
   - [ ] `input_number.mev_escalation_delay_minutes` (2-10 min, default 5)
   - [ ] `input_number.mev_deescalation_delay_minutes` (5-20 min, default 10)
   - [ ] `input_number.mev_rate_window_minutes` (2-10 min, default 5)

5. **Template Sensors:**
   - [ ] `sensor.first_floor_humidity_rate` (%/min, rolling window)
   - [ ] `sensor.mev_air_quality_demand` (%, from CO₂/IAQ)
   - [ ] `sensor.mev_humidity_demand` (%, from position + rate)
   - [ ] `sensor.mev_final_fan_speed` (%, max of AQ and humidity demands)

---

## Technical Implementation

### Method 1: UI-Based Creation

Create via **Settings → Devices & Services → Helpers**.

### Method 2: YAML-Based Creation

Create `packages/mev_helpers.yaml` and reference in `configuration.yaml`:

```yaml
homeassistant:
  packages:
    mev_helpers: !include packages/mev_helpers.yaml
```

**Full Helper Configuration:**

See Epic 16 Brief (Story 16.5 section) for complete YAML configuration including:
- All 10 input_number entities
- 1 input_select entity
- 4 template sensors

Key input_numbers:
- `mev_humidity_upper_bound` / `mev_humidity_lower_bound` (window bounds)
- `mev_escalation_fan_threshold` / `mev_escalation_fan_threshold_cooling` (escalation triggers)
- `mev_deescalation_fan_threshold` (de-escalation trigger)
- `mev_escalation_delay_minutes` / `mev_deescalation_delay_minutes` (timing)
- `mev_rate_window_minutes` (rate calculation)
- `mev_minimum_fan_speed` (baseline ventilation)
