## User Story

As a **system integrator**,  
I want **all required Home Assistant helper entities created and configured**,  
So that **the MEV control logic in ESPHome has the runtime-configurable parameters it needs**.

---

## Story Context

This story consolidates all HA helper entity creation. These helpers are imported by the ESPHome MEV component to allow remote tuning without reflashing.

**Entity Types:**
- **Input Numbers:** Runtime-configurable parameters (humidity bounds, fan thresholds, delays)
- **Input Select:** Handled by ESPHome, but visible in HA.

---

## Acceptance Criteria

### Required Helper Entities (to be created in HA)

1. **Humidity Window:**
   - [x] `input_number.mev_humidity_upper_bound` (30-90%, default 60%)

   - [x] `input_number.mev_escalation_fan_threshold` (50-90%, default 70%)
   - [x] `input_number.mev_deescalation_fan_threshold` (10-50%, default 30%)

3. **Timing (for transition delays):**
   - [x] `input_number.mev_escalation_delay_minutes` (1-15 min, default 5)
   - [x] `input_number.mev_deescalation_delay_minutes` (5-30 min, default 10)
   - [x] `input_number.mev_rate_window_minutes` (2-15 min, default 5)
