# Story 8.5: Production Rollout - Remaining Rooms Migration

**Epic:** 8 - Unified State Machine Architecture  
**Story Type:** Production Migration  
**Estimated Effort:** 14 days (1 room/day × 6 rooms + buffer)  
**Priority:** High (Production Deployment)  
**Date:** October 31, 2025

---

## User Story

As a **system owner**,  
I want **all remaining production rooms (6 rooms) migrated from Epic 5/7 to Epic 8 architecture following a phased rollout schedule**,  
So that **the entire system benefits from the unified coordinator pattern with reduced code duplication and improved extensibility**.

## Story Context

**Existing System Integration:**

- **Completed:** Soggiorno test room migration validated (Story 8.4) ✅
- **Target rooms:** 6 remaining production rooms across 3 tiers
- **Current state:** All rooms using Epic 5 + Epic 7 components
- **Migration approach:** Phased rollout with 1 room/day cadence
- **Technology:** ESPHome YAML packages, Git version control

  - Device YAMLs: `devices/distribuzione-primo-piano.yaml`, `devices/distribuzione-piano-terra.yaml`

## Risk and Compatibility Check

**Primary Risk:** Multiple room migrations compound failure probability

**Mitigation:**
- Staggered 1-room/day cadence isolates issues
- Each room independently validated before proceeding
- Tier 1 (critical rooms) migrated first with extra validation
- Tier 2 (lower-priority) migrated only after Tier 1 stable

**Secondary Risk:** HA automation errors cause PID stuck in OFF

**Mitigation:**
- Test all HA automations before rollout
- Manual PID override available via HA dashboard
- Coordinator status sensor provides clear diagnostics

**Tertiary Risk:** Performance degradation across multiple devices

**Mitigation:**
- Monitor ESP32 free heap on all devices during rollout
- Watch for coordinator interval delays (should be steady 5s)
- Test room validates performance acceptable before rollout

**Rollback Procedure (Per Room):**

```bash
# If any room fails validation, immediate rollback:
git checkout epic-7-{room_name}-baseline -- components/rooms/{floor}/{room_name}.yaml
esphome compile devices/{device_name}.yaml
esphome upload devices/{device_name}.yaml
# Rollback time: < 10 minutes per room
```

**Stop-Gate Criteria (Halt Rollout):**

- ❌ Any room requires rollback after 12h soak test
- ❌ Climate quality degrades >2°C variance in any room
- ❌ Systematic errors detected (affects multiple rooms)
- ❌ Performance issues (memory leaks, coordinator delays)
- ❌ User-reported comfort issues in migrated rooms

**Compatibility Verification:**

- [ ] Epic 5/7 and Epic 8 rooms coexist without conflicts during rollout
- [ ] No shared global variable collisions
- [ ] ESP32 devices on same network operate independently
- [ ] HA can handle mixed Epic 5/7/8 entity names during transition
