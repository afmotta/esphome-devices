## Story

**As a** system operator,  
**I want** the existing 3-tier sensor failover component to work correctly for room temperature sensors from ESP32 devices,  
**so that** room PIDs can use UDP room sensors as primary source with HA fallback, enabling autonomous operation during HA outages.

---

## Acceptance Criteria

1. ✅ Component `components/sensor_failover_3tier.yaml` exists and implements 3-tier failover: UDP → HA → NaN
2. ✅ Component accepts required vars: `zone_slug`, `zone_name`, `udp_sensor_id`, `ha_temperature_sensor_id`
3. ✅ Component accepts optional var: `emergency_timeout` (default: 180s)
4. ✅ Component does NOT create UDP sensor internally - expects caller to provide pre-existing UDP sensor ID
5. ✅ Component creates HA sensor (Tier 2) from `ha_temperature_sensor_id`
6. ✅ Component creates abstracted sensor `${zone_slug}_room_temp_abstracted` for PID use
7. ✅ Tier 1 (UDP) is active when UDP sensor has valid value (!isnan)
8. ✅ Tier 2 (HA) is active when UDP is NaN but HA sensor has valid value
9. ✅ Tier 3 (Emergency) returns NaN when both UDP and HA are unavailable
10. ✅ Component creates diagnostic text_sensor `${zone_slug}_sensor_tier` showing "UDP"/"HA"/"Emergency"
11. ✅ Tier transitions are logged with ESP_LOG* showing zone name and transition reason
12. ✅ Component header documentation clearly explains Epic 11 usage pattern
13. ✅ Component compiles successfully (syntax validation)
14. ✅ Component is backward compatible with Epic 9 supply temperature pattern

### Key Changes from Epic 9 to Epic 11

**Epic 9 (Supply Temperature) Usage:**
- Mixing board UDP broadcasts supply temp
- Distribution boards receive via packet_transport
- **Problem:** Component tried to create UDP sensor internally (doesn't work with packet_transport)

**Epic 11 (Room Temperature) Usage:**
- External ESP32 room sensor UDP broadcasts room temp
- Distribution board receives via packet_transport (created in device YAML)
- Component references pre-existing UDP sensor by ID
- **Solution:** Component expects UDP sensor already created, just references it

### Component Interface Contract

**Caller Responsibilities (Device/Room YAML):**
1. Create UDP sensor using `packet_transport` platform:
   ```yaml
   sensor:
     - platform: packet_transport
       id: soggiorno_udp_temp
       provider: soggiorno_room_sensor
       internal: true
   ```

2. Include sensor_failover_3tier with vars:
   ```yaml
   sensors: !include
     file: ../../sensor_failover_3tier.yaml
     vars:
       zone_slug: "soggiorno"
       zone_name: "Soggiorno"
       udp_sensor_id: "soggiorno_udp_temp"
       ha_temperature_sensor_id: "sensor.room_soggiorno_temperature"
   ```

**Component Responsibilities:**
- Create HA sensor (Tier 2)
- Create abstracted sensor with 3-tier failover logic
- Create diagnostic text_sensor
- Log tier transitions

### Variable Naming Standards

**Required Variables:**
- `zone_slug`: Identifier for IDs (lowercase, underscores) - e.g., "soggiorno", "bagno_grande"
- `zone_name`: Display name for logs/sensors (title case) - e.g., "Soggiorno", "Bagno Grande"
- `udp_sensor_id`: ID of pre-existing UDP sensor (created by caller)
- `ha_temperature_sensor_id`: HA entity ID for fallback (e.g., "sensor.room_soggiorno_temperature")

**Optional Variables:**
- `emergency_timeout`: Seconds before emergency trigger (default: 180)

**Generated IDs (by component):**
- `${zone_slug}_ha_temperature_tier2`: HA sensor (internal)
- `${zone_slug}_room_temp_abstracted`: Abstracted sensor for PID use (internal)
- `${zone_slug}_sensor_tier`: Diagnostic text_sensor (exposed to HA)
